## 1. 文档目标

本文档用于把“作品工作台”从概念落成 Electron 桌面端可实现的页面原型与交互流转设计，重点回答：

1. 作品工作台在当前菜单结构中如何落位。
2. 工作台首页应该展示哪些信息。
3. 从“生成”到“检查”到“发布”如何在桌面端一站式流转。
4. 作品详情页、版本面板、检查弹窗、发布入口如何协同。
5. 如何在不推翻当前 Task 页面体系的前提下平滑迁移。

---

## 2. 设计结论先行

### 2.1 核心结论

建议新增一个 **作品工作台（Creative Workbench）** 作为桌面端的主业务入口，并逐步替代“任务列表”承担的产品主视图职责。

### 2.2 页面定位

- **任务列表**：保留，逐步下沉为执行视图 / 诊断视图。
- **作品工作台**：新增，作为业务视图 / 内容视图 / 发布准备视图。
- **任务详情页**：保留，作为作品详情页中的“执行面板”或跳转诊断页。

### 2.3 一句话解释

- 操作者平时盯的是 **作品工作台**。
- 只有出现合成/发布/调度问题时，才下钻看 **任务详情**。

## 3. 在现有 Electron 桌面结构中的落位建议

## 3.1 当前导航现状

当前左侧菜单主要分为：

- 数据看板
- 账号管理
- 任务管理
- 素材中心
- AI 剪辑
- 系统设置

当前“任务管理”下包含：

- 任务列表
- 创建任务
- 调度配置
- 配置档管理

## 3.2 推荐调整后导航结构

建议在“任务管理”中引入作品层：

```text
任务管理
  ├─ 作品工作台   ← 新主入口
  ├─ 创建任务
  ├─ 任务列表     ← 下沉为执行视图
  ├─ 调度配置
  └─ 配置档管理
```

### 推荐路由

- `/creative/workbench`
- `/creative/:id`
- `/creative/:id/version/:versionId`（可选）

### 兼容策略

- 第一阶段保留 `/task/list`
- 新增 `/creative/workbench`
- 作品详情页内可跳转到相关 task 详情

## 4. 页面体系总览

建议增加以下 4 个页面/交互单元：

1. **作品工作台页**
2. **作品详情页**
3. **版本历史面板**
4. **人工检查弹窗 / 抽屉**

此外保留对现有页面的衔接：

- `TaskCreate` 作为生成入口
- `TaskDetail` 作为执行诊断页
- `ProductList / ProductDetail` 作为来源素材入口
- `AIClip` 作为独立工具页，后续接入生成流

---

## 5. 页面一：作品工作台

## 5.1 页面目标

作品工作台是桌面端主业务入口，统一承接：

- 查看作品生成结果
- 进入人工检查
- 发起局部重生成
- 筛选“可发布作品池”
- 进入发布动作

## 5.2 页面布局建议

```text
┌──────────────────────────────────────────────┐
│ 顶部工具栏：搜索 / 筛选 / 批量操作 / 快捷入口 │
├──────────────────────────────────────────────┤
│ 左侧筛选栏（可折叠）                          │
│  - 状态                                       │
│  - 来源商品                                   │
│  - 配置档                                     │
│  - 发布账号                                   │
│  - 最近失败                                   │
├──────────────────────────────────────────────┤
│ 主区：作品卡片列表 / 表格列表切换             │
│  - 默认卡片                                   │
│  - 支持批量选择                               │
├──────────────────────────────────────────────┤
│ 底部状态栏：选中数量 / 可发布数量 / 失败数量   │
└──────────────────────────────────────────────┘
```

## 5.3 顶部工具栏

建议包含：

- 搜索框（标题 / 商品名 / 编号）
- 状态筛选器
- 视图切换（卡片 / 表格）
- 批量操作：
  - 批量标记可发布
  - 批量标记需调整
  - 批量发布
- 快捷入口：
  - 创建任务
  - 打开调度中心
  - 打开 AIClip

## 5.4 卡片结构

每张作品卡片建议包含：

### 视觉区

- 封面缩略图
- 视频预览按钮
- 状态角标（如：待检查 / 可发布 / 发布失败）

### 文本区

- 作品标题 / 商品名
- 文案摘要（2 行截断）
- 话题数量 / 简要展示
- 来源商品名
- 当前版本号

### 状态区

- 当前作品状态
- 最近失败信息（若有）
- 最近检查结果
- 最近更新时间

### 操作区

- 检查
- 编辑
- 重生成
- 发布
- 更多

## 5.5 表格视图字段建议

| 字段 | 说明 |
|------|------|
| 作品编号 | creative_no |
| 标题 | 当前标题 |
| 来源商品 | product name |
| 当前版本 | version_no |
| 状态 | creative status |
| 检查结果 | 最近检查结果 |
| 发布账号 | 默认账号 |
| 最近失败 | 错误摘要 |
| 更新时间 | updated_at |
| 操作 | 查看 / 检查 / 发布 |

## 6. 页面二：作品详情页

## 6.1 页面目标

作品详情页用于承接：

- 当前版本内容展示
- 人工检查
- 版本切换
- 局部返工
- 执行状态查看

## 6.2 页面布局建议

```text
┌──────────────────────────────────────┐
│ Header：返回 / 标题 / 状态 / 快捷操作 │
├──────────────────────────────────────┤
│ 左：预览区                            │
│  - 视频播放器                         │
│  - 封面图                             │
├──────────────────────────────────────┤
│ 右：信息与动作区                      │
│  - 标题                               │
│  - 文案                               │
│  - 话题                               │
│  - 检查结论                           │
│  - 可发布性提示                       │
│  - 操作按钮                           │
├──────────────────────────────────────┤
│ 下：版本历史 / 执行记录 Tabs          │
└──────────────────────────────────────┘
```

## 6.3 Header 操作建议

- 返回工作台
- 标记可发布
- 标记需调整
- 放弃作品
- 立即发布
- 更多：
  - 打开关联 Task
  - 打开本地文件目录
  - 复制错误信息

## 6.4 右侧信息区

建议至少包含：

- 当前版本号
- 当前标题
- 当前文案
- 当前话题
- 当前配置档
- 当前发布账号
- 成品完整性状态
- direct publish / composition 语义提示

## 6.5 下方 Tabs 建议

### Tab 1：版本历史

展示：

- 版本号
- 生成原因
- 创建时间
- 检查结论
- 是否当前版本
- 是否已发布

### Tab 2：执行记录

展示：

- 关联 Task 列表
- 合成状态
- 发布状态
- quick retry / edit retry 记录

### Tab 3：日志与文件

展示：

- 错误日志摘要
- 最终视频路径
- 封面路径
- 文案文件路径
- 打开目录按钮

## 7. 交互单元一：人工检查弹窗 / 抽屉

## 7.1 打开时机

- 从工作台点击“检查”
- 从详情页点击“检查结论”
- 当作品进入 `WAITING_REVIEW` 时可自动引导打开

## 7.2 弹窗结构建议

### 顶部

- 当前作品编号
- 当前版本号
- 当前状态

### 中部

- 视频预览
- 文案预览
- 封面缩略图
- 检查清单（可选勾选）

### 底部动作

- 确认可发布
- 需调整
- 放弃
- 取消

### 备注区

- 检查意见文本框
- 返工方向选择（当点击“需调整”时展开）

## 7.3 返工方向选项建议

- 重生成文案
- 重合成视频
- 替换封面
- 修改话题
- 手工编辑并另存新版本

---

## 8. 交互单元二：版本历史面板

## 8.1 展示目标

让操作者看清：

- 当前版本从何而来
- 哪些版本被检查过
- 哪些版本被发布过
- 最近失败的是哪一版

## 8.2 列表字段建议

| 字段 | 说明 |
|------|------|
| 版本号 | v1 / v2 / v3 |
| 类型 | generated / manual_edit / rerendered_video |
| 原因 | 初次生成 / 重生成文案 / 替换封面 |
| 检查结果 | approved / rework_required / rejected |
| 发布时间 | 若有 |
| 当前版本 | yes/no |
| 操作 | 设为当前 / 对比 / 打开详情 |

## 8.3 版本对比建议

初期不一定做复杂 diff，但至少提供：

- 文案变化摘要
- 封面变化前后
- 视频文件路径差异
- 话题列表差异

## 9. 页面间流转

## 9.1 主流转路径

```text
商品/素材准备
  → 创建任务
  → 生成作品
  → 进入作品工作台
  → 打开作品详情
  → 人工检查
  → 标记可发布
  → 进入发布池
  → 发布成功/失败
```

## 9.2 返工流转路径

```text
作品工作台
  → 检查
  → 标记需调整
  → 选择返工方向
  → 生成新版本
  → 回到作品详情
  → 再次检查
```

## 9.3 失败流转路径

```text
发布失败
  → 工作台显示失败标签
  → 打开作品详情
  → 查看执行记录 / 错误日志
  → 决定 quick retry 或重新返工
```

## 10. 与现有页面的迁移关系

## 10.1 TaskList 的角色变化

当前：

- 主任务入口
- 业务和执行混合视图

后续：

- 下沉为执行列表
- 更多用于运维/调试/失败排查

## 10.2 TaskDetail 的角色变化

当前：

- 任务主详情页

后续：

- 保留为“执行详情页”
- 从作品详情中的执行记录 Tab 跳转进入

## 10.3 TaskCreate 的角色变化

当前：

- 直接创建任务

后续：

- 仍作为生成入口
- 生成完成后默认跳转到作品工作台，而不是只回任务列表

## 11. Electron 桌面交互细节建议

## 11.1 适合桌面端的交互

建议强化：

- 双栏布局
- 抽屉式细节面板
- 右键菜单（可选）
- 打开本地目录
- 打开文件
- 拖拽替换封面/视频（后续可选）

## 11.2 桌面端提示建议

对于关键动作，建议统一 toast + 持久状态提示：

- 已创建新版本
- 已标记为可发布
- 已标记为需调整
- 发布失败，点击查看详情
- 本地文件缺失，请重新生成

## 11.3 桌面端异常展示优先级

优先展示：

1. 当前作品是否可发布
2. 当前版本是否完整
3. 最近一次执行失败原因
4. 本地文件是否存在
5. 是否受授权状态影响

## 12. 页面状态与权限逻辑（单人桌面模式）

本地无多人角色切换，但仍有状态门禁。

### 作品工作台按钮规则建议

| 状态 | 检查按钮 | 发布按钮 | 重生成按钮 |
|------|----------|----------|------------|
| WAITING_REVIEW | 可用 | 不可用 | 可用 |
| APPROVED | 可用 | 可用 | 可用 |
| REWORK_REQUIRED | 可用 | 不可用 | 可用 |
| PUBLISHING | 禁用 | 禁用 | 禁用 |
| PUBLISHED | 可用（只读） | 可再次发布（可选） | 可用 |
| REJECTED | 只读 | 不可用 | 不可用 |

## 14. 一句话定义

> 作品工作台是桌面端真正的业务主界面：它把“作品内容、人工检查、版本历史、发布准备和执行诊断”统一到一个连续工作流中，而不是让操作者在任务页、素材页和工具页之间来回切换。
