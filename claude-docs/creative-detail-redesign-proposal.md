# CreativeDetail 精简设计方案

> 日期：2026-04-25
> 基于：`creative-detail-user-confirmation.md`

---

## 一、设计原则

### 1.1 一句话原则

> **页面只做一件事：让用户快速完成作品定义并提交生成。**

### 1.2 核心原则

1. **A 区是唯一主角**：页面视觉重心放在入选区，其他都是来源
2. **实时反馈**：B/C 区操作立即在 A 区体现，无需保存
3. **简洁优先**：不要让用户看到不需要的信息
4. **操作闭环**：每个来源操作都能直接采用到 A 区

### 1.3 不包含的内容

- ~~D 区（版本/审核）~~
- ~~E 区（发布诊断）~~
- ~~legacy editor~~
- ~~大量 Alert 提示~~

这些功能要么移到其他页面，要么折叠到深层入口。

## 二、页面结构

### 2.1 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  Header                                                         │
│  [← 返回]  作品名称                           [状态]  [主 CTA]  │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  A 区：当前入选区（主区域，占据页面 60% 高度）                   │
│  ├─ 商品名称                                                    │
│  ├─ 封面                                                        │
│  ├─ 文案                                                        │
│  ├─ 音频                                                        │
│  └─ 视频集合                                                    │
│                                                                 │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  来源区（Tab 切换：B 商品 / C 自由素材）                         │
│  [商品] [自由素材]                                              │
│                                                                 │
│  Tab 内容：候选素材列表                                          │
│  每项：[采用] [预览]                                            │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 为什么不左右分栏

| 左右分栏 | Tab 切换 |
|---------|---------|
| B/C 同时可见，内容多，页面宽 | 同一时刻只展示一个来源，内容集中 |
| 需要用户同时扫描两列 | 用户聚焦一个来源后，再切 Tab |
| 标题/卡片重复 | 标题不重复，节省空间 |

## 三、Header 设计

### 3.1 精简 Header

```
┌────────────────────────────────────────────────────────────────┐
│ [← 返回]  夏日轻氧唇釉作品              [部分就绪]  [提交生成] │
│            最近更新：22:40                    缺：封面、音频    │
└────────────────────────────────────────────────────────────────┘
```

包含：
- 返回入口
- 作品名称（可编辑）
- 状态标签（草稿 / 部分就绪 / 可生成）
- 主 CTA（提交生成）
- 辅助信息（最近更新时间、缺失项）

### 3.2 不在 Header 展示

- ~~Alert 提示~~ → 改为内联提示
- ~~大量次 CTA~~ → 收纳到 `...` 菜单
- ~~readiness 摘要文字~~ → 用"缺：封面、音频"代替

## 四、A 区设计（入选区）

### 4.1 A 区是唯一主角

A 区占据页面主要空间（60%+），是用户最常看的区域。

### 4.2 A 区卡片结构

```
┌────────────────────────────────────────────────────────────────┐
│ 当前入选                                                         │
│────────────────────────────────────────────────────────────────│
│                                                                 │
│  商品名称  ████████████████████████████████████████████        │
│  来源：主题商品 / 手动编辑                    [编辑] [恢复]   │
│                                                                 │
│  封面      ████████                                  [更换]   │
│  来源：自由素材                                        [预览]   │
│                                                                 │
│  文案      「今夏轻氧上唇，一抹显色...」            [编辑]     │
│  来源：文案生成                                      [展开]     │
│                                                                 │
│  音频      轻快电商节奏 BGM 03                      [更换]     │
│  来源：自由素材                                        [试听]   │
│                                                                 │
│  视频集合  ████ ████ ████ ████                      [管理]   │
│  共 4 条                                                    [添加]  │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 4.3 每张卡片的交互

| 字段 | 操作 | 来源标签 | 缺失时 |
|------|------|---------|--------|
| 商品名称 | 编辑 / 恢复默认 | 显示 | 显示"未设定" |
| 封面 | 更换 / 预览 | 显示 | 显示"待选择" + 快速入口 |
| 文案 | 编辑 / 展开全文 | 显示 | 显示"待选择" |
| 音频 | 更换 / 试听 | 显示 | 显示"待选择" |
| 视频集合 | 管理 / 添加 | 显示列表 | 显示"暂无入选" + 添加入口 |

### 4.4 视频集合卡片

视频是集合位，需要特殊处理：

```
┌─ 视频集合 ──────────────────────────────────────────────────┐
│                                                              │
│  [视频1缩略]  开场  来源：商品  [×移除]                    │
│  [视频2缩略]  主镜头  来源：自由  [×移除]                   │
│  [视频3缩略]  CTA  来源：商品  [×移除]                     │
│                                                              │
│  [添加视频 +]                                               │
│                                                              │
│  ── 或点击 [管理视频] 打开抽屉编辑 ──                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 5.4 操作语义统一

| 类型 | 操作按钮 | 说明 |
|------|---------|------|
| 封面（唯一位） | [采用] | 点击后 A 区封面立即更新 |
| 音频（唯一位） | [采用] | 点击后 A 区音频立即更新 |
| 文案（唯一位） | [采用] | 点击后 A 区文案立即更新 |
| 视频（集合位） | [入选] / [移出] | 点击后 A 区视频集合立即更新 |

**无保存按钮**：每个操作立即生效。

## 六、缺失项处理

### 6.1 A 区缺失时

```
┌─ 封面 ──────────────────────────────────────────────────────┐
│                                                              │
│  ☐ 待选择                                                    │
│  [去商品区选择]  [从自由素材选择]                             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

缺失项不是空白卡片，而是带快速入口的占位卡。

### 6.2 快速补位

点击缺失项的入口，直接切换到来源 Tab 并聚焦对应分类。

## 七、状态感知

### 7.1 三种状态

| 状态 | Header 展示 | A 区 | 来源区 |
|------|------------|------|--------|
| 未开始 | "草稿" | 显示缺失项占位卡 | 折叠（未选商品时） |
| 部分就绪 | "部分就绪" + "缺：X 项" | 部分卡片有值，部分缺失 | 展开 |
| 可生成 | "可生成" | 全部有值 | 可操作 |

### 7.2 主 CTA

```
[提交生成]  ← 只有"可生成"状态时可用
```

## 八、与原设计的对比

| 项目 | 原设计 | 新设计 |
|------|--------|--------|
| 区域数 | A/B/C/D/E 五区 | A/B/C 三区 |
| B/C 分栏 | 左右分栏 | Tab 切换 |
| 保存方式 | 批量保存 | 实时生效 |
| legacy editor | 保留兼容 | 直接废弃 |
| Hero 复杂度 | 多个 Alert + 大量按钮 | 精简：标题 + 状态 + 主 CTA |
| D/E 区 | 首屏可见 | 移除 |

## 九、实现优先级

### Phase 1：核心闭环
1. Header 精简（去除 Alert）
2. A 区卡片结构重建
3. B/C Tab + 实时采用操作
4. 删除 legacy editor

### Phase 2：体验优化
1. 视频集合管理抽屉
2. 缺失项快速入口
3. 状态切换动画

---

## 十、验收标准

1. 用户打开页面，3 秒内知道"这是哪个作品、现在能不能提交"
2. 用户从商品 Tab 选一个封面，1 秒内看到 A 区封面更新
3. 用户不需要找"保存"按钮
4. 页面不显示不需要的信息（D/E/Alert）
5. B/C 区不是左右分栏，而是 Tab 切换
