# CreativeDetail 交互设计确认纪要

> 确认日期：2026-04-25
> 确认人：用户

---

## 用户确认的设计意图

### 1. 操作节奏：实时生效

**选择：实时生效**

B/C 区的素材操作（设为当前封面、加入入选视频等）立即在 A 区有反馈，不需要单独点"保存"。

实现路径：每个操作 → API PATCH → A 区立即刷新。

---

### 2. 页面布局：精简为 A/B/C 三区

**选择：去掉 D/E 区**

页面只需要三个区：
- A 区（入选区）：核心，实时更新
- B 区（商品区）：来源
- C 区（自由素材区）：来源

版本结果、发布诊断等功能不在这个页面承载。

---

### 3. B/C 区分栏方式：不是左右分栏

**用户明确：不喜欢 B/C 左右分栏的设计。**

具体形式待定，可能是：
- 纵向堆叠
- Tab 切换
- 折叠面板

---

### 4. legacy editor：废弃

**选择：B，直接废弃**

在 P0-3~P0-6 把区内操作全部闭环，然后删除 legacy editor。

不再保留"高级编辑模式"。

---

### 5. 整体诉求：简洁、易用

用户的核心感受：

> 当前页面设计，根本就不便捷，没有考虑易用性。

具体问题：
- Hero + 大量 Alert 占据首屏
- 页面太满，不知道看哪里
- 操作路径不清晰

---

## 设计意图更新

基于以上确认，设计意图更新为：

```
页面只有三个区：
  A 区（入选区）：唯一主角，实时更新，承载所有生成输入
  B 区（商品区）：来源区，提供候选，支持快速采用到 A 区
  C 区（自由素材区）：来源区，提供候选，支持快速采用到 A 区

关系：
  B/C 区操作 → A 区实时刷新
  无需保存按钮

不包含：
  D 区（版本/审核）
  E 区（发布诊断）
  legacy editor
```
