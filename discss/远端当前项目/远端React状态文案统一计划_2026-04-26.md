# 远端 React 状态文案统一计划（2026-04-26）

## 本次目标
- 统一 `remote-admin` 全站的：
  - 空状态（empty state）
  - 错误状态（error state）
  - Danger zone / destructive action 提示
- 保持现有功能、路由、测试合同不变，只做文案与轻量复用收口。

## 当前不一致点
- 空状态：
  - 有的用 `No ... matched the current filters.`
  - 有的用 `Choose ... from the list ...`
  - 有的用 `Select ... to inspect detail.`
- 错误状态：
  - 有的写 `The ... could not be loaded from the admin API.`
  - 有的写 `The selected ... request failed.`
  - dashboard 还用了另一套更偏实现细节的表述。
- Danger zone：
  - users / devices / sessions 的标题相同，但描述句式不同；
  - 只读角色提示有的写 `view detail only`，有的写 `inspect`，口径不统一。

## 本次统一规则
1. **空状态**
   - 过滤结果为空：`No <resource> matched the current filters.`
   - 未选择列表项：`Select <item> from the list to inspect <detail>.`
2. **错误状态**
   - 会话缺失：`Please sign in again to load <resource>.`
   - 列表失败：`The <resource> list could not be loaded from the admin API.`
   - 详情失败：`The selected <resource> detail could not be loaded from the admin API.`
3. **Danger zone**
   - 保留标题 `Danger zone`
   - 写操作描述统一强调：
     - 审计
     - 密码确认
     - 短时 step-up token
     - 成功后的后端刷新
   - 只读描述统一用：
     - `This role can review <resource>, but <actions> remain disabled.`

## 实施方式
- 补一个轻量 shared copy helper，收敛空状态 / 错误状态句式。
- 页面内的 Danger zone 文案直接按统一模板收口。

## 验收标准
- `remote/remote-admin/src/pages/**` 中相关状态文案遵循同一语气和结构
- typecheck / build / test / react-step-up 回归通过

## 实施结果
- 新增 `remote/remote-admin/src/components/states/adminCopy.ts`
  - 抽出会话缺失、通用加载失败、列表失败、详情失败、过滤为空、列表选择为空等文案 helper
- 已收口页面：
  - `dashboard/DashboardPage.tsx`
  - `users/UsersPage.tsx`
  - `devices/DevicesPage.tsx`
  - `sessions/SessionsPage.tsx`
  - `audit/AuditLogsPage.tsx`
- 统一后的口径：
  - 空状态：`No <resource> matched the current filters.`
  - 选择为空：`Select <item> from the list to inspect <detail>.`
  - 会话缺失：`Please sign in again to load <resource>.`
  - 加载失败：`The <resource> / <resource> list / selected <resource> detail could not be loaded from the admin API.`
  - Danger zone：
    - 写操作统一强调 **server-side audit + password confirmation + short-lived step-up token + backend refresh**
    - 只读统一强调 **This role can review <resource>, but <actions> remain disabled.**

## 风险评估
- 本次为纯文案与轻量复用收口：
  - 不改接口合同
  - 不改交互流程
  - 不改路由
  - 不引入新依赖
- 现有 React 级 step-up 回归未对这些文案做强耦合断言，因此本次调整风险低。
