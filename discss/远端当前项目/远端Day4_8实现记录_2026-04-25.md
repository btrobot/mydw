# 远端 Day 4.8 实现记录（OpenAPI response 声明与 route contract 对齐）

## 目标

在 **不改运行时行为** 的前提下，把远端后端 OpenAPI 中的 response 声明收敛到统一入口，并补一层自动化回归，避免后续 route contract 已变更但文档声明漂移。

## 本次落地

### 1. 抽出统一的 OpenAPI error response helper

新增：

- `remote/remote-backend/app/api/openapi_responses.py`

提供：

- `error_responses(*status_codes)`

统一收敛当前远端 API 已在使用的标准错误响应声明：

- `401 -> Unauthorized`
- `403 -> Forbidden`
- `404 -> Not Found`
- `429 -> Too Many Requests`

所有错误响应仍然指向同一个 `ErrorResponse` schema。

### 2. 将 auth / admin / self-service 路由切到统一声明入口

更新：

- `remote/remote-backend/app/api/auth.py`
- `remote/remote-backend/app/api/admin.py`
- `remote/remote-backend/app/api/self_service.py`

这次改动只收口 decorator 里的 `responses=...` 声明：

- 不改 handler 逻辑
- 不改 service 逻辑
- 不改状态码
- 不改错误 body
- 不改前端合同

### 3. 新增 OpenAPI route response contract 回归测试

新增：

- `remote/remote-backend/tests/api/test_openapi_response_contract.py`

覆盖范围：

- auth：
  - `/login`
  - `/refresh`
  - `/logout`
  - `/me`
- self-service：
  - `/self/me`
  - `/self/devices`
  - `/self/sessions`
  - `/self/activity`
  - `/self/sessions/{session_id}/revoke`
- admin：
  - `/admin/login`
  - `/admin/session`
  - `/admin/users`
  - `/admin/users/{user_id}`（GET / PATCH）
  - `/admin/users/{user_id}/revoke`
  - `/admin/users/{user_id}/restore`
  - `/admin/devices`
  - `/admin/devices/{device_id}`
  - `/admin/devices/{device_id}/unbind`
  - `/admin/devices/{device_id}/disable`
  - `/admin/devices/{device_id}/rebind`
  - `/admin/sessions`
  - `/admin/sessions/{session_id}/revoke`
  - `/admin/audit-logs`
  - `/admin/metrics/summary`

回归内容包括：

- `200` 成功 schema 引用正确
- `401 / 403 / 404 / 429` 的 `ErrorResponse` schema 引用正确
- 错误描述与当前 OpenAPI 标准描述一致
- 需要校验参数 / body 的路由继续包含 `422 Validation Error`

## 风险判断

这是一次 **文档声明 + 测试回归** 收口：

- 不触碰运行时返回内容
- 不改变现有 contract 语义
- 主要风险是 decorator 声明在收口时漏掉某个 route 的状态码

这次通过“全量关键 route 的 OpenAPI contract 测试”把这个风险压回测试层，而不是把风险留给手工比对。

## 为什么这是最小风险 Day 4.8

1. **不改真实 API 行为**
   - 只改 OpenAPI 声明组织方式
   - 行为层仍由既有 Day 4.1 ~ Day 4.7 回归保护

2. **把声明漂移风险从人工检查变成自动化校验**
   - 后续如果有人改了 `responses=...`
   - 测试会直接提示是哪个 route 漂移

3. **和 Day 4.7 的 helper 收口方向一致**
   - Day 4.7 收口的是 route contract 测试样板
   - Day 4.8 收口的是 OpenAPI response 声明样板
