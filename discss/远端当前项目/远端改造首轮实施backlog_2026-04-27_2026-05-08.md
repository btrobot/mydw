# 远端改造首轮实施 Backlog（2026-04-27 ～ 2026-05-08）

日期：2026-04-25

## 1. 适用范围

这份 backlog 是对《远端改造可执行任务清单》的**首轮实施拆解**，目标是用 **2 周**时间，把远端改造推进到“可安全开工、可稳定迁第一页”的状态。

**本轮只覆盖：**

1. Slice 0：冻结现状行为与改造边界
2. Slice 1：补后端 service 测试地板
3. Slice 2：补 RBAC + API 最小集成测试
4. Slice 3：搭新远端前端骨架
5. Slice 4：迁移 login + dashboard
6. Slice 5：users 页只做到**可启动**，不承诺本轮必须收尾

**本轮明确不做：**

- Refine 选型切换
- 分页 shape 全量统一
- Alembic
- 密码哈希升级
- step-up security

## 3. 本轮总目标（Definition of Done）

到 **2026-05-08** 为止，本轮完成定义如下：

1. 已形成一版现状行为基线与迁移验收清单
2. 后端 admin service 核心动作有基础自动化测试
3. RBAC 与关键 API 错误码路径有最小集成测试
4. 新远端前端 React 骨架可运行
5. login + dashboard 在新前端可跑通
6. users 页至少完成数据接入与页面骨架，不一定要求全部动作闭环

如果第 6 条做不到，但前 1～5 条全部完成，本轮仍算**基本成功**

## 10. 本轮失败信号

出现下面任一情况，说明本轮节奏需要立刻收缩：

1. B1/B2 长时间做不完，说明后端测试切面拆得太大
2. B3 还没起好就开始改 users/devices/sessions，说明前端在抢跑
3. B4 迁移 login 时顺手改后端协议，说明边界失守
4. B5 想同时做列表抽象、分页统一、动作闭环，说明 scope 已膨胀
