# 远端启动脚本迁移入口修复记录（2026-04-26）

## 背景

`scripts/start-remote.bat` 在 “Bootstrapping remote admin account” 阶段仍调用：

```bat
python -c "from app.migrations.runner import upgrade; upgrade()"
```

但 `remote/remote-backend` 已在 Day 5.2 切到 Alembic 作为唯一受支持的迁移入口，`app.migrations.runner` 已不再存在，因此启动脚本会直接报：

```text
ModuleNotFoundError: No module named 'app.migrations.runner'
```

## 本次修复

- 删除 `start-remote.bat` 中失效的 legacy runner 导入调用。
- 保留并复用当前受支持的入口：

```bat
python scripts/bootstrap_admin.py --migrate ...
```

`bootstrap_admin.py --migrate` 内部已经调用 `app.migrations.alembic.ensure_database_on_head()`，可以同时覆盖：

1. 空库升级到 Alembic `head`
2. 旧库 baseline stamp + upgrade
3. admin 账户幂等创建 / 更新

## 为什么这是最小风险修复

- 不回滚到已废弃的 runner 兼容层
- 不新增新的脚本入口
- 直接对齐 `remote/remote-backend/README.md` 当前文档约定
- 避免“先迁移一次，再在 bootstrap 中重复迁移一次”的冗余路径

## 验证点

- 单独执行 `python scripts/bootstrap_admin.py --migrate ...` 能完成迁移与 admin bootstrap
- `scripts/start-remote.bat` 可越过原来的 `ModuleNotFoundError` 故障点
