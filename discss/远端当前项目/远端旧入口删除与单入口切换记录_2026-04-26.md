# 远端旧入口删除与单入口切换记录（2026-04-26）

## 目标

根据最新决定，`remote-admin` 不再保留“默认入口 + 旧入口 fallback”双入口结构，而是直接切到 **单一 React 入口**。

本次目标不是删除所有历史代码，而是删除 **用户可见的旧入口路径**，避免继续出现：

- 修复在新入口上完成
- 用户却从旧入口进入
- 最终形成“代码和看到的页面不是一套”的错位

## 本次切换方式

采用最小风险切换：

1. **保留现有 React 应用代码与构建产物路径**
2. **把仓库根 `index.html` 直接提升为 React 开发入口**
3. **删除 `react-index.html`**
4. **构建后的唯一静态入口统一为 `dist-react/index.html`**
5. **脚本、README、测试全部指向这一个入口**

## 具体改动

### 1. 入口文件

- `remote/remote-admin/index.html`
  - 从 legacy 静态壳切换为 React 入口
  - `<script type="module" src="/src/react-main.tsx">`

- `remote/remote-admin/react-index.html`
  - 已删除

### 2. Vite 构建

- `vite.config.ts`
  - 不再显式指定 `react-index.html` 为输入
  - 让 `index.html` 成为默认 Vite 入口
  - 构建产物输出到 `dist-react/index.html`

### 3. 脚本与文档

已同步更新：

- `scripts/start-remote.bat`
- `scripts/status-all.bat`
- `remote/remote-admin/README.md`

现在默认打开的是：

- `http://127.0.0.1:4173/dist-react/index.html?apiBase=http://127.0.0.1:8100`

## 为什么这次不删 legacy 源码

这次用户要求删除的是“旧入口”，不是马上清空所有 legacy 代码。

当前仍保留：

- `src/main.ts`
- `src/app/App.ts`
- 以及依赖它们的现有 node:test 覆盖

原因是：

1. 这样可以把“用户入口切换”与“历史代码清仓”拆成两刀
2. 先收口用户入口，风险最低
3. 后续若要真正删除 legacy 实现，可单独迁移或替换相关测试

## 验证口径

这次切换后，应以以下路径作为唯一入口验证：

- 开发态：`/index.html`
- 静态构建态：`/dist-react/index.html`

后续所有 UI 修复默认都应针对这条单入口链路进行验证。
