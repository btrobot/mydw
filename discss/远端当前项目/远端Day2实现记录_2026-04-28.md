# 远端 Day 2 实现记录

日期：2026-04-28

## 本次落地范围

本次实现对应首周执行单中的 Day 2，目标是把远端 React 新壳真正立起来，同时不破坏现有 legacy admin shell 和现有测试基线。

## 已完成

1. 在 `remote/remote-admin` 中引入 React + TypeScript + Vite + Ant Design + React Query + React Router
2. 保留 legacy `index.html + dist/main.js` 路线不动
3. 新增 `react-index.html` 作为 React 壳入口
4. 新增 React 基础目录与文件：
   - `src/react-main.tsx`
   - `src/app/providers.tsx`
   - `src/app/AppShell.tsx`
   - `src/routes/router.tsx`
   - `src/features/auth/auth-client.ts`
   - `src/features/auth/auth-store.ts`
   - `src/features/auth/auth-context.tsx`
   - `src/pages/login/LoginPage.tsx`
   - `src/pages/dashboard/DashboardPage.tsx`
   - `src/components/states/*`
5. 新壳已接上现有 admin backend：
   - login
   - session restore
   - dashboard metrics
6. users / devices / sessions / audit 暂时挂占位页，明确仍在迁移队列中

## 兼容策略

这次没有直接重写：

- `remote/remote-admin/src/main.ts`
- `remote/remote-admin/src/app/App.ts`

而是采用“**legacy 保留，新 React 壳并行起步**”的方式推进，这样可以：

1. 保留现有 node:test 回归资产
2. 保留旧 admin shell 作为 fallback
3. 让后续页面迁移按页切换，而不是一次性爆炸替换

## 运行方式

### Legacy

```bash
npm run test
```

或按 README 方式 build 后打开 `index.html`

### React 新壳

```bash
npm run dev
```

浏览器将打开：

```text
/react-index.html
```
