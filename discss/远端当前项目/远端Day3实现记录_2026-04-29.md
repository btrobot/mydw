# 远端 Day 3 实现记录（auth/route guard 边界 + users 列表页迁移起步）

## 本次落地范围

- 补稳 React 新壳下的 auth 过期处理边界：
  - session restore 失败时给出可关闭提示；
  - dashboard / users 请求命中 `token_expired` 时统一回退登录页；
  - 登录页承接过期提示，避免“静默掉回匿名态”。
- 恢复并重建 `AppShell.tsx`，补回 Day 2 中断后缺失的受保护布局。
- 将 `/users` 从占位页切到真实 React 页面，先落地“列表 + 过滤 + 详情”最小可用闭环。

## 具体实现

### 1. auth 边界

- `auth-context.tsx`
  - 新增 `authNotice`
  - 新增 `handleExpiredSession()`
  - 新增 `clearAuthNotice()`
- `login/LoginPage.tsx`
  - 展示并允许关闭 `authNotice`
- `dashboard/DashboardPage.tsx`
  - 遇到 `token_expired` 时清理本地 session 并跳转 `/login`

### 2. 路由与受保护壳

- 重建 `app/AppShell.tsx`
  - 恢复受保护布局；
  - 补齐 `/dashboard`、`/users`、`/devices`、`/sessions`、`/audit-logs` 导航；
  - header 中展示当前管理员身份、角色与登出入口。
- `routes/router.tsx`
  - `/users` 由 `PendingPage` 切换为 `UsersPage`

### 3. users 列表页迁移（最小风险版）

- 新增 `pages/users/UsersPage.tsx`
  - 使用 React Query 请求 `/admin/users` 与 `/admin/users/{id}`
  - 提供 search / status / license filters
  - 左侧列表 + 右侧详情 rail
  - 明确区分只读角色与可写角色，但**本 slice 不接 revoke / restore / patch**
- `auth-client.ts`
  - 补齐 users 相关类型与 API helper：
    - `AdminUserRecord`
    - `AdminUserListResponse`
    - `AdminUsersFilters`
    - `buildUsersQuery()`
    - `getAdminUsers()`
    - `getAdminUserDetail()`
    - `isAuthExpiredError()`
    - `canEditUsersRole()`

## 为什么这是“最小风险”

1. **先补壳，再迁页**
   - 先恢复 `AppShell`，避免 Day 2 中断状态继续扩大影响面。
2. **先迁只读闭环，不碰 destructive mutation**
   - 先验证列表、筛选、详情、过期跳转是否稳定；
   - 把 revoke / restore 留到下一 slice，避免 Day 3 同时引入写操作风险。
3. **优先统一 session 过期回退**
   - 这部分是所有后续页面迁移的共同前置条件；
   - 提前补稳可以减少后续 devices / sessions / audit 的重复处理。

## 下一步建议

- 明确 mutation 成功后的列表/详情刷新策略
- 补充 role-based action disable 状态文案

### Slice 3.2
- 迁移 devices 列表页
- 复用相同的：
  - token 过期回退
  - 左右栏布局

## 通过标准

- 未登录访问受保护路由仍被拦截
- session 过期时，dashboard / users 能稳定回退登录页
- `/users` 不再是占位页
- users 列表可筛选、可选中、可查看详情
- 只读角色文案清晰，但不会误暴露可写操作
