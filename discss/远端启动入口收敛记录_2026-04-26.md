# 远端启动入口收敛记录（2026-04-26）

## 背景

此前 `scripts/start-remote.bat` 默认打开的是 legacy 入口：

- `index.html?apiBase=http://127.0.0.1:8100`

但近期左侧导航视觉修复、step-up 交互、React 级回归测试，主要都落在 React 壳上。这样会导致一个明显错位：

- 开发/修复在 React 壳进行
- 用户/一键启动却打开 legacy 壳

结果就是：修复看起来“没效果”，因为实际打开的不是同一套 UI。

## 本次收敛策略

本次不做激进切换，不删除 legacy，只把 **默认入口** 收敛清楚：

1. `start-remote.bat` 默认打开 React 静态产物入口
2. legacy 入口保留为显式 fallback
3. `status-all.bat` 和 `remote-admin/README.md` 同步到同一套默认口径

## 具体改动

### 1. 一键启动默认 URL

从：

- `http://127.0.0.1:4173/index.html?apiBase=http://127.0.0.1:8100`

改为：

- `http://127.0.0.1:4173/dist-react/react-index.html?apiBase=http://127.0.0.1:8100`

同时额外打印 legacy fallback：

- `http://127.0.0.1:4173/index.html?apiBase=http://127.0.0.1:8100`

### 2. 一键启动构建链路

为了保证默认入口可用，`start-remote.bat` 现在会顺序执行：

- `npm run build`
- `npm run build:react`

前者继续保留 legacy fallback，后者产出 React 默认入口 `dist-react/react-index.html`。

### 3. 文档与状态口径同步

同步更新：

- `scripts/status-all.bat`
- `remote/remote-admin/README.md`

这样本地状态查看、一键启动提示、README 手动运行说明，三者不再各说各话。

## 风险控制

这次没有：

- 删除 legacy 壳
- 改动后端接口
- 改动 auth 合同
- 改动 React 或 legacy 的业务页面逻辑

只是把“默认打开哪一个入口”收口到已经承担主线演进的 React 壳上。

## 后续建议

如果后面继续推进远端 admin：

- 所有新的 UI/交互修复默认应以 React 壳为主入口验证
- legacy 壳只作为明确 fallback，而不是继续与默认入口并列
