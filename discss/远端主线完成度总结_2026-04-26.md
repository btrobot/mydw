本轮“远端最小风险改造”主线已经完成闭环，原计划 Slice 0 ~ Slice 11 均已落地，没有仍处于半完成状态的主线项。

## 二、已完成范围

### 1. 前端迁移主线

- remote-admin React 新壳、route guard、登录态恢复机制已落地
- login、dashboard、users、devices、sessions、audit logs 页面已完成迁移
- destructive 流程已统一接入确认、刷新与错误处理闭环
- legacy fallback 在迁移阶段保持可用，降低切换风险

### 2. 后端契约与测试补稳

- RBAC 与关键管理接口最小集成测试已补齐
- destructive API 的 401 / 403 / 404 / 429 最小 contract 入口已建立
- `/refresh`、`/logout`、`/me`、self-service 关键 route 的最小回归入口已补齐
- admin 与 self-service 列表返回 shape 已统一为：
  - `items`
  - `total`
  - `page`
  - `page_size`

### 3. 基础设施与风险收口

- 登录限流替换方案已落地，且优先保持前端合同不变
- Alembic baseline 已建立，迁移入口已统一
- Argon2 平滑升级方案已完成设计与实现：
  - 双栈校验
  - opportunistic rehash

### 4. admin step-up security 已闭环

已完成：

- 后端 step-up schema / route / service enforcement
- destructive admin route 的 `X-Step-Up-Token` 合同
- admin 前端 destructive 流程接入 step-up verify + token header
- 浏览器级回归覆盖：
  - users revoke / restore
  - devices disable / unbind / rebind
  - sessions revoke

## 三、完成价值

这轮主线完成后，项目得到的核心收益是：

### 1. 风险更低

- 迁移遵循“小步替换、合同稳定、逐页闭环”的路线
- 没有一次性重写大块核心逻辑
- destructive 能力新增了 step-up 二次确认，安全边界明显提升

### 2. 可维护性更高

- 分页与列表 shape 收口，前后端消费关系更一致
- route contract test helper 和最小回归入口让后续改动更容易守住边界
- Alembic 成为唯一迁移入口，后续 schema 演进更可控

### 3. 可验证性更高

- 关键后端 contract 已有最小回归护栏
- 关键 destructive 前端流程已有浏览器级 step-up 回归
- 迁移后的页面不再只靠“人工点点看”，而是有明确回归矩阵支撑
