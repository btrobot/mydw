# 远端 Day 3.4 实现记录（audit logs 迁移）

## 本次 slice 目标

- 将 `/audit-logs` 从 React shell 中的占位页迁移为真实可用页面。
- 继续遵守“最小风险改造”原则：**不改后端契约、不引入新依赖、不做乐观本地写回**。
- 复用 Day 3 已验证的路由守卫与 token 过期处理方式。

## 实现范围

### 1. 补齐 audit client 契约

在 `remote/remote-admin/src/features/auth/auth-client.ts` 中新增：

- `AdminAuditRecord`
- `AdminAuditListResponse`
- `AdminAuditFilters`
- `toUtcAuditFilterTimestamp()`
- `buildAuditLogQuery()`
- `getAdminAuditLogs()`

约束保持与旧实现一致：

- 查询参数顺序保持稳定：
  1. `event_type`
  2. `actor_id`
  3. `target_user_id`
  4. `target_device_id`
  5. `target_session_id`
  6. `created_from`
  7. `created_to`
  8. `limit`
  9. `offset`
- `datetime-local` 输入在发往后端前转换为 UTC ISO 字符串：
  - `created_from` 对齐到 `:00.000`
  - `created_to` 对齐到 `:59.999`

### 2. 新增 React audit 页面

新增 `remote/remote-admin/src/pages/audit/AuditLogsPage.tsx`：

- 支持筛选：
  - event type
  - actor id
  - target user/device/session
  - created from / to
  - limit / offset
- 支持 query preview，便于对照真实后端请求。
- 支持上一页 / 下一页。
- 采用“**列表即事实来源**”模式：
  - 无单独 detail API
  - 详情面板直接从当前列表项派生
  - 翻页或筛选后若当前选中项失效，则自动回退到第一页第一条
- 详情区保留旧页面的重点信息：
  - actor / user / device / session tags
  - request id / trace id / created at
  - 高亮 `reason` / `required_permission` / `user_id` / `device_id` / `session_id`
  - 剩余 payload 以 JSON 展示

### 3. 路由接线

更新 `remote/remote-admin/src/routes/router.tsx`：

- 移除 `/audit-logs` 的 `PendingPage`
- 直接接入 `AuditLogsPage`

## 风险控制评价

这次改造的风险较低，主要因为：

1. **只读页面**：没有新增 mutation，避免了 users/devices/sessions 那类“动作成功但刷新失败”的一致性问题。
2. **后端真相优先**：筛选、分页、详情都以现有后端返回为准，没有本地 cache 拼装逻辑。
3. **契约对齐旧实现**：query 参数顺序、时间边界、详情字段展示都沿用 legacy 语义。
4. **鉴权边界复用既有模式**：请求报 `token_expired` 时统一走 `handleExpiredSession()` + 跳回 `/login`。

## 仍需关注

- 当前 audit detail 仍然是“列表项派生详情”，如果未来后端为单条 audit 增加更完整 detail API，再考虑升级。
- `created_from / created_to` 依赖浏览器本地时区解释 `datetime-local`，这与 legacy 行为一致，但跨时区排查时要注意。
- 目前没有为 React 页面新增 UI 级交互测试，主要依赖 typecheck / build / 现有 node test 做回归保护。
