> 日期：2026-04-24  
> 适用范围：CreativeDetail / PRD 收敛阶段  
> 目的：把已有来源文档中的关键结论，映射到未来主 PRD 的章节，作为收敛与查漏的工作矩阵  
> 状态：Draft

---

## 1. 使用说明

这份矩阵不是主 PRD，  
而是主 PRD 的**覆盖检查表**。

用途：

1. 写主 PRD 前，先知道每条关键结论该放到哪一章
2. 写主 PRD 后，逐条回查有没有遗漏
3. 避免关键规则只存在于局部讨论文档里

矩阵字段说明：

- **来源编号**：对应《CreativeDetail PRD 来源总表 v1》
- **关键结论**：必须被主 PRD 承接的规则
- **建议进入 PRD 章节**：主 PRD 中的落位
- **纳入状态**：
  - `待纳入`
  - `已纳入`
  - `不纳入（显式弃用）`

当前版本先全部标成 `待纳入`。

## 2. 建议的主 PRD 章节骨架

建议未来主 PRD 至少包含以下章节：

1. 页面定位与目标

4. 输入合同（唯一位 / 集合位）
5. 默认值与控制权规则

7. 自由素材区规则
8. 当前入选区规则
9. 状态 / 页面模式 / 结果确认流
10. 版本 / 草稿 / 发布边界
11. CTA 与关键操作
12. P0 范围与非目标

## 4. 最容易漏掉的关键项

主 PRD 编写时，建议重点盯以下 8 项，因为最容易漏：

1. 主题商品名称仅在作品名称为空时默认回填  
2. 用户手改后自动脱钩  
3. 视频是集合位，不是唯一位  
4. 音频若只能一个，应按唯一位处理  
5. 回到编辑不创建新版本  
6. 发布不创建新版本  
7. 商品选择不是普通 Select  
8. 三分区的职责边界

## 5. 主 PRD 封版前检查清单

主 PRD 初稿完成后，必须至少检查：

- [ ] 来源总表里的主来源是否都至少映射了一条关键结论
- [ ] 矩阵里是否还有“待纳入”但无归宿的核心规则
- [ ] 默认值规则是否完整写入
- [ ] 用户控制权与脱钩规则是否完整写入
- [ ] 版本创建边界是否完整写入
- [ ] 三分区职责是否完整写入
- [ ] 当前入选区字段合同是否完整写入

只要这些未完成，就不应把主 PRD 视为封版。

## 6. 使用建议

推荐工作顺序：

1. 先确认《来源总表》范围
2. 用本矩阵作为写作蓝图
3. 写《CreativeDetail 收敛 PRD v1》
4. 写完后把本矩阵的“待纳入”逐条改成“已纳入”或“显式弃用”

## 7. 最终结论

这份覆盖矩阵的意义在于：

> **把“前面已经讨论过的很多正确结论”，变成主 PRD 编写时可核对、可回查、可证明不遗漏的收敛工具。**

也就是说，主 PRD 之后不是“感觉写全了”，而是：

> **能用矩阵证明写全了。**
