## 0. 文档目的

这份文档不是视觉稿，也不是实现稿。

它的目的只有一个：

> **把已经收敛的 CreativeDetail PRD，翻译成“整页怎么摆、每区放什么、主次关系是什么、按钮放哪、状态怎么表现”的低保真页面蓝图。**

这份线框文案版用于回答：

- 首屏先给用户看什么？
- 三个核心区怎么排？
- “当前入选”怎么成为页面第一主角？
- “商品区 / 自由素材区”怎么作为来源区存在，而不抢主焦点？
- 结果确认、版本、发布诊断放在页面哪里，才不会打断定义流程？

---

## 1. 页面设计总原则

## 1.1 页面一句话

> **CreativeDetail 是一个“定义作品最终生成输入”的工作台页面。**

不是任务页，不是纯发布页，也不是单纯素材库页。

---

## 1.2 整页排序原则

整页必须遵守下面的优先级：

1. **先看当前最终入选了什么**
2. **再看这些内容从哪里来**
3. **再看是否已经可以生成**
4. **最后才看结果确认、版本、发布跟进**

也就是：

> **定稿优先，来源其次，结果跟进后置。**

---

## 1.3 页面结构原则

整页采用：

- **首屏摘要 + 当前入选区**
- **中段来源区（商品区 / 自由素材区）**
- **下段结果与发布跟进区**

避免：

- 一上来就铺满复杂表单
- 一上来就铺满版本和发布信息
- 让来源区抢走定稿区的视觉主导权

---

## 1.4 UI 倾向（吸收 ui-ux-pro-max 的使用结论）

这页更适合：

- 工作台式结构
- 卡片化分区
- 清晰层级
- 强状态提示
- 明确 CTA

不适合：

- 落地页式大 Hero
- 纯表单堆叠
- 过重视觉装饰
- 结果区过早压到首屏

---

## 2. 整页信息架构总览

```text
页面头部
├─ 返回 / 页面标题 / 作品基础状态 / 最近更新时间
├─ 主 CTA / 次 CTA / 高级 CTA
└─ readiness 摘要条（缺什么、是否可生成）

首屏主区
└─ A. 当前入选区（页面第一主角）
   ├─ 商品名称
   ├─ 封面
   ├─ 文案
   ├─ 音频
   └─ 视频集合

中段来源区
├─ B. 商品区（主题商品 + 来源候选）
└─ C. 自由素材区（补充来源）

下段跟进区
├─ D. 结果确认 / 当前版本区
└─ E. 发布诊断 / 发布记录区
```

---

## 3. 页面整体骨架（桌面版）

## 3.1 页面头部区

### 目标

让用户在进入页面 3 秒内知道：

- 我现在在定义哪个作品
- 这个作品当前处于什么阶段
- 还差什么才能生成
- 下一步最正确的动作是什么

### 线框文案

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 返回作品列表                                                        │
│ CreativeDetail / 作品名称                                           │
│ 状态：未开始 / 部分就绪 / 可生成 / 结果待确认 / 已发布              │
│ 最近更新：xx:xx    当前主题商品：已选择 / 未选择                    │
│----------------------------------------------------------------------│
│ 缺失：封面 1 项 / 音频 1 项 | 当前已选视频：3 条 | 当前版本：V3      │
│                                                                      │
│ [主 CTA：提交生成] [次 CTA：保存草稿] [次 CTA：返回列表]             │
│ [高级 CTA：更多操作 ▼]                                               │
└──────────────────────────────────────────────────────────────────────┘
```

### 说明

- 页面标题区不需要复杂副导航。
- 状态、缺失项、最近更新时间必须在头部可见。
- “提交生成”只有在可生成时高亮可用。
- “更多操作”里承接低频动作，例如：
  - 重新命名作品
  - 复制作品
  - 查看历史版本
  - 删除草稿（若业务允许）

---

## 3.2 readiness 摘要条

### 目标

把“还差什么”和“为什么暂时不能生成”说清楚。

### 线框文案

```text
┌ readiness 摘要条 ────────────────────────────────────────────────────┐
│ 当前状态：部分就绪                                                  │
│ 要生成本作品，还缺：音频、封面                                      │
│ 建议操作：先去商品区补默认封面，或去自由素材区选择素材             │
└──────────────────────────────────────────────────────────────────────┘
```

### 规则

- **未开始**：几乎没有核心输入
- **部分就绪**：已有部分输入，但未达到生成门槛
- **可生成**：核心输入已满足
- **结果待确认 / 已发布**：不再用 readiness 语言主导，而切换为结果状态语言

---

## 4. A. 当前入选区（首屏第一主区）

## 4.1 区块目标

回答一个核心问题：

> **如果现在立刻提交生成，系统到底会拿什么去生成？**

因此它必须是首屏第一主角。

---

## 4.2 区块结构

```text
┌ A. 当前入选区 ─ 当前作品最终生成输入 ───────────────────────────────┐
│ 说明：这里展示当前真正会用于生成的内容。来源可能来自商品区或自由素材区 │
│                                                                      │
│ [商品名称卡] [封面卡]                                                │
│ [文案卡（宽）]                                                       │
│ [音频卡]                                                             │
│ [视频集合卡（多条）]                                                 │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 4.3 商品名称卡

### 线框文案

```text
┌ 商品名称 ─ 当前生效 ────────────────────────────────────────────────┐
│ 名称：夏日轻氧唇釉礼盒                                               │
│ 来源：主题商品 / 手动编辑                                            │
│ 状态：已定义                                                         │
│ [编辑名称] [恢复主题商品名称]                                        │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互要求

- 必须显式显示当前生效值。
- 必须显式显示来源。
- 若是手动编辑过，显示“已脱钩”提示。

---

## 4.4 封面卡

### 线框文案

```text
┌ 封面 ─ 当前生效 ────────────────────────────────────────────────────┐
│ [封面缩略图]                                                         │
│ 来源：主题商品 / 自由素材                                            │
│ 状态：已定义 / 缺失                                                  │
│ [替换封面] [查看大图] [恢复默认]                                     │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互要求

- 缺失时不能只显示空白，必须显示缺失提示和补位入口。
- “替换封面”动作应跳转到对应来源区的选择上下文，而不是弹出完全脱离上下文的上传框。

---

## 4.5 文案卡

### 线框文案

```text
┌ 文案 ─ 当前生效 ────────────────────────────────────────────────────┐
│ 文案摘要：                                                          │
│ 「今夏轻氧上唇，一抹显色，持久通勤不黏腻……」                        │
│ 来源：文案生成机制 / 手动编辑 / 外部导入                             │
│ [编辑文案] [重新生成文案] [展开全文]                                 │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互要求

- 文案在当前入选区必须被当作“唯一位”处理。
- 即使文案由外部机制生成，也要在当前入选区显式承接。

---

## 4.6 音频卡

### 线框文案

```text
┌ 音频 ─ 当前生效 ────────────────────────────────────────────────────┐
│ 音频：轻快电商节奏 BGM 03                                            │
│ 来源：自由素材 / 默认模板                                            │
│ 状态：已定义 / 缺失                                                  │
│ [更换音频] [试听] [取消采用]                                          │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互要求

- 音频按唯一位处理。
- 若缺失，但业务上允许后补，也要明确显示“当前不可生成/仍缺音频”。

---

## 4.7 视频集合卡

### 线框文案

```text
┌ 视频集合 ─ 当前已入选 3 条 ─────────────────────────────────────────┐
│ [视频1缩略卡] [来源标签] [取消采用]                                  │
│ [视频2缩略卡] [来源标签] [取消采用]                                  │
│ [视频3缩略卡] [来源标签] [取消采用]                                  │
│ [管理已选视频]                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互要求

- 视频集合不使用“唯一主视频”的表达。
- 这里重点展示“当前实际入选了哪些视频”。
- 若顺序对生成有意义，应允许排序。

---

## 4.8 当前入选区底部操作条

### 线框文案

```text
┌ 当前入选区底部操作条 ───────────────────────────────────────────────┐
│ 当前状态：部分就绪                                                  │
│ 缺失：音频、封面                                                    │
│ [去商品区补默认] [去自由素材区补充] [提交生成]                       │
└──────────────────────────────────────────────────────────────────────┘
```

### 设计意图

让用户在看完当前入选之后，立刻知道下一步是“补”还是“生成”。

## 5. B. 商品区（来源区 1）

## 5.1 区块目标

回答：

> **主题商品是谁？它给当前作品带来了哪些默认候选？当前又采用了哪些？**

---

## 5.2 商品区整体结构

```text
┌ B. 商品区 ─ 主题商品与默认来源 ────────────────────────────────────┐
│ 当前主题商品：已选择 / 未选择                                        │
│ [选择商品] [更换商品] [查看商品详情摘要]                             │
│----------------------------------------------------------------------│
│ 已选主题商品摘要卡                                                    │
│ - 封面                                                               │
│ - 标题 / 商品名称                                                    │
│ - 视频数量                                                           │
│ - 基础信息                                                           │
│ [展开查看更多素材]                                                   │
│----------------------------------------------------------------------│
│ 商品来源素材区                                                       │
│ - 商品名称候选（单选采用）                                           │
│ - 封面候选（单选采用）                                               │
│ - 视频候选（多选采用）                                               │
│ - 标题/文案候选（若有）                                              │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 5.3 商品选择入口

### 线框文案

```text
┌ 商品选择入口 ───────────────────────────────────────────────────────┐
│ 当前主题商品：唇釉礼盒 / 未选择                                      │
│ [选择商品] [更换商品]                                                │
│ 辅助说明：选择主题商品后，可自动带入封面、名称、视频等默认候选      │
└──────────────────────────────────────────────────────────────────────┘
```

### 交互模式

商品选择不应是简单下拉框。

推荐模式：

- 点击“选择商品”
- 打开商品选择器（抽屉 / 弹层）
- 以“封面 + 视频 + 标题 + 商品名称”的摘要卡形式浏览
- 选择完成后回填页面

---

## 5.4 已选主题商品摘要卡

### 线框文案

```text
┌ 已选主题商品摘要卡 ────────────────────────────────────────────────┐
│ [商品封面]                                                           │
│ 商品名称：夏日轻氧唇釉礼盒                                           │
│ 标题：轻氧显色，日常通勤一抹完成                                     │
│ 视频：4 条    封面：1 张                                              │
│ [展开素材] [查看商品信息]                                            │
└──────────────────────────────────────────────────────────────────────┘
```

### 设计要求

- 选完商品后，用户必须能在页内回看其核心信息。
- 不应要求用户再次离开页面查看商品详情。

---

## 5.5 商品来源素材列表

### 线框文案

```text
┌ 商品来源素材列表 ───────────────────────────────────────────────────┐
│ 商品名称候选                                                         │
│ [✓] 使用该商品名称：夏日轻氧唇釉礼盒                                 │
│                                                                      │
│ 封面候选                                                             │
│ [✓] 封面 A（当前已采用） [预览]                                      │
│ [ ] 封面 B [采用为当前封面]                                          │
│                                                                      │
│ 视频候选                                                             │
│ [✓] 视频 1（当前已入选）                                             │
│ [✓] 视频 2（当前已入选）                                             │
│ [ ] 视频 3                                                           │
│ [✓] 视频 4（当前已入选）                                             │
│                                                                      │
│ 标题 / 文案候选（若有）                                              │
│ [✓] 采用标题作为文案起点 / [转为文案候选]                            │
└──────────────────────────────────────────────────────────────────────┘
```

### 规则表达

- 商品名称、封面是**唯一位勾选**
- 视频是**集合位勾选**
- 每条候选都要显式标明：
  - 是否已被当前作品采用
  - 采用后会影响哪个当前入选字段

## 6. C. 自由素材区（来源区 2）

## 6.1 区块目标

回答：

> **除了主题商品带来的默认素材之外，还有哪些自由维护素材被补充进当前作品？**

---

## 6.2 区块结构

```text
┌ C. 自由素材区 ─ 手动补充来源 ──────────────────────────────────────┐
│ [上传素材] [从素材库选择] [批量导入]                                 │
│----------------------------------------------------------------------│
│ 封面素材                                                             │
│ 音频素材                                                             │
│ 视频素材                                                             │
│ 文案/文本素材                                                        │
│----------------------------------------------------------------------│
│ 每条素材都要显示：                                                   │
│ - 基础摘要                                                           │
│ - 来源：自由素材                                                     │
│ - 是否已被当前作品采用                                               │
│ - 采用到哪个当前入选字段                                             │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 6.3 自由素材列表单元

### 封面类

```text
│ [✓ 采用为当前封面] [预览] [删除]                                      │
```

### 视频类

```text
┌ 自由视频素材 B ─────────────────────────────────────────────────────┐

│ 来源：自由素材                                                       │
│ 状态：已入选 / 未入选                                                │
│ [✓ 纳入当前视频集合] [预览] [删除]                                    │
└──────────────────────────────────────────────────────────────────────┘
```

### 音频类

```text
┌ 自由音频素材 C ─────────────────────────────────────────────────────┐
│ 音频名：快节奏带货 BGM                                               │
│ 来源：自由素材                                                       │
│ 状态：已采用为当前音频 / 未采用                                       │
│ [✓ 采用为当前音频] [试听] [删除]                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 文案类

```text
┌ 自由文案素材 D ─────────────────────────────────────────────────────┐

│ [✓ 采用为当前文案] [编辑] [删除]                                      │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 6.4 自由素材区设计要求

- 自由素材区不是“杂物堆”。
- 必须按素材类型分组。
- 必须标清“是否已采用”。
- 必须让用户一眼看懂：这个素材现在有没有真正进入作品定义。

---

## 7. D. 结果确认 / 当前版本区（下段）

## 7.1 区块目标

回答：

> **最近一次生成产出了什么？当前处于哪个版本？用户是否确认采用该结果？**

这个区必须后置，不能抢首屏。

---

## 7.2 区块结构

```text
┌ D. 结果确认 / 当前版本区 ──────────────────────────────────────────┐
│ 当前版本：V3                                                         │
│ 状态：待确认 / 已确认 / 已发布                                       │
│ 生成时间：xx:xx                                                      │
│----------------------------------------------------------------------│
│ [结果预览卡]                                                         │
│ - 封面 / 片段 / 文案摘要 / 音频                                      │
│----------------------------------------------------------------------│
│ [确认采用结果] [回到编辑] [重新生成] [查看版本记录]                  │
└──────────────────────────────────────────────────────────────────────┘
```

### 设计要求

- 只有当至少生成过一个版本后，这个区才完整出现。
- 未生成前，只显示轻量占位说明，不占过大面积。
- “重新生成”是创建新版本的明确入口。

---

## 8. E. 发布诊断 / 发布记录区（最下段）

## 8.1 区块目标

回答：

> **当前版本是否适合发布？发布过几次？最近一次发布情况如何？**

这是跟进区，不是首要定义区。

---

## 8.2 区块结构

```text
┌ E. 发布诊断 / 发布记录区 ──────────────────────────────────────────┐
│ 当前版本：V3                                                         │
│ 发布适配状态：可发布 / 有风险 / 未确认结果                           │
│----------------------------------------------------------------------│
│ 发布前检查                                                           │
│ [✓] 已有封面                                                         │
│ [✓] 已有文案                                                         │
│ [✓] 已有视频                                                         │
│ [ ] 当前结果尚未确认                                                 │
│----------------------------------------------------------------------│
│ 最近发布记录                                                         │
│ - 2026-04-23 18:20 发布到渠道 A                                      │
│ - 2026-04-22 16:05 发布到渠道 B                                      │
│----------------------------------------------------------------------│
│ [发起发布] [查看发布记录]                                            │
└──────────────────────────────────────────────────────────────────────┘
```

### 设计要求

- 发布能力默认压在页面最下段。
- 不要在首屏给“发布”比“提交生成”更强的存在感。
- 若结果未确认，发布 CTA 应显式受限。

---

## 9. 主 CTA / 次 CTA / 高级 CTA 体系

## 9.1 主 CTA

主 CTA 永远围绕当前主任务切换：

- 编辑定义阶段：`提交生成`
- 结果确认阶段：`确认采用结果`
- 发布阶段：`发起发布`

但同一时刻只允许一个主 CTA 最强。

---

## 9.2 次 CTA

常见次 CTA：

- 保存草稿
- 回到编辑
- 查看版本记录
- 去补素材

这些动作必须存在，但不应抢主 CTA。

---

## 9.3 高级 CTA

收纳到“更多操作”：

- 复制作品
- 重命名作品
- 删除草稿
- 查看调试信息
- 导出信息（若后续需要）

---

## 10. 页面模式切换下的线框变化

## 10.1 编辑定义模式

页面重点：

- A 当前入选区
- B 商品区
- C 自由素材区

D/E 降低存在感。

---

## 10.2 结果待确认模式

页面重点变为：

- 头部状态条切换为“结果待确认”
- D 区上移强化
- A 区仍保留，用于让用户比对“当前输入”和“当前结果”
- B/C 进入弱可编辑态或折叠态

---

## 10.3 已发布跟进模式

页面重点：

- 顶部提示“当前版本已发布”
- E 区展示最近发布记录
- A 区仍作为当前定义快照存在
- 若允许继续修改，则通过“回到编辑 / 重新生成”显式开启下一轮

---

## 11. 状态设计（loading / error / empty / readiness / locked）

## 11.1 Loading

### 页面级 loading

```text
正在加载作品定义、当前版本、来源素材……
```

要求：

- 保留头部骨架
- A/B/C 使用骨架屏
- 不要让页面整体跳闪

---

## 11.2 Error

### 页面级 error

```text
作品信息加载失败
[重试] [返回列表]
```

### 区块级 error

例如商品素材加载失败时：

```text
商品来源素材加载失败
[重试加载]
```

---

## 11.3 Empty

### 未选择主题商品

```text
你还没有选择主题商品
选择后可自动带入商品名称、封面、视频等默认候选
[选择商品]
```

### 自由素材为空

```text
还没有添加自由素材
[上传素材] [从素材库选择]
```

### 尚未生成版本

```text
还没有生成结果
完成当前定义后，可提交生成首个版本
[提交生成]
```

---

## 11.4 Readiness

### 未开始

```text
你还没有开始定义这个作品
建议先选择主题商品
```

### 部分就绪

```text
作品定义已完成一部分
还缺：封面、音频
```

### 可生成

```text
当前作品定义已满足生成条件
现在可以提交生成新版本
```

---

## 11.5 Locked / 弱可编辑态

当页面处于结果确认或某些处理中状态时：

- A 区允许查看
- B/C 区可进入弱可编辑态
- 通过显式按钮进入“回到编辑”

不要出现“看起来能改，其实不能改”的伪可编辑态。

---

## 12. 移动端收敛建议

移动端不重做逻辑，只改变排布：

### 顺序

1. 页面头部
2. readiness 摘要
3. A 当前入选区
4. B 商品区
5. C 自由素材区
6. D 结果确认 / 当前版本区
7. E 发布诊断 / 发布记录区

### 规则

- A 区各字段卡片纵向堆叠
- 视频集合采用横滑卡片或折叠列表
- 商品选择器优先使用全屏抽屉
- 主 CTA 吸底，但不遮挡关键信息

---

## 14. 这份线框稿向 PRD 回写什么

这份线框稿确认后，建议回写进 PRD 的内容有：

- 首屏必须先出现当前入选区
- 来源区放在中段，不抢首屏主焦点
- 结果确认 / 发布跟进放在下段
- 页面头部必须带 readiness 摘要和缺失项
- 当前入选区必须显式展示来源与当前生效值
- 主 CTA 按页面主任务切换，但同屏只允许一个最强主 CTA

不建议整份全文并回 PRD。

## 15. 收口结论

一句话收口：

> **CreativeDetail 的整页低保真骨架，应当是“头部状态摘要 + 首屏当前入选区 + 中段双来源区 + 下段结果与发布跟进区”的工作台结构。**

再压缩成一句：

> **先定稿，再追来源，最后处理结果与发布。**
