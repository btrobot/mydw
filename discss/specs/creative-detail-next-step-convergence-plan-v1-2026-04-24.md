# 《CreativeDetail 下一步推进与收敛建议 v1》

> 日期：2026-04-24  
> 适用范围：CreativeDetail / 当前设计推进阶段  
> 目的：回答“接下来如何推进？是先收敛成完整 PRD 再重做 UI，还是别的方式？”  
> 状态：Draft

---

## 1. 一句话结论

> **现在最需要的不是继续发散讨论，也不是立刻重做 UI，更不是直接写代码。**

当前最正确的推进方式是：

> **先收敛，再出整页蓝图，再切实现切片。**

也就是按下面顺序走：

1. **收敛成一份 CreativeDetail 主 PRD**
2. **基于 PRD 产出整页低保真 UI 蓝图**
3. **把蓝图拆成 P0 / P1 / P2 实现切片**
4. **只先做 P0**

---

## 2. 你现在为什么会迷惑

不是你判断错了，而是当前已经积累了很多“局部正确”的文档：

- 状态机
- 单用户简化
- 版本机制
- 商品选择
- 商品名称与素材勾选
- 入选区字段
- 三分区页面结构

这些内容单看都成立，  
但如果没有一个“总收敛文档”，就会出现典型问题：

### 2.1 你会不知道哪个文档才是主文档

### 2.2 你会不知道现在讨论的是页面规范、业务规则，还是实现细节

### 2.3 你会不知道下一步应该继续设计，还是开始做

所以你现在的迷惑，本质上不是“想不清楚”，而是：

> **已经到了必须从“多篇分析文档”进入“单一主线设计文档”的节点。**

---

## 3. 现在不建议怎么做

## 3.1 不建议继续无边界发散讨论

因为再继续加新话题，会让已有结论越来越难收口。

---

## 3.2 不建议立刻上手重做完整 UI

因为现在虽然规则越来越清楚，但还没有一份真正统一的主规范。

如果直接重做 UI，后面很容易遇到：

- 页面结构改一半又推翻
- 字段语义变了，组件也得跟着重改
- P0/P1 混在一起做

---

## 3.3 不建议直接进入实现

因为现在已经不再是“小修小补”，而是在改：

- 页面心智
- 状态模型
- 字段合同
- 三分区结构

这已经是一个中等规模重构，不适合“边写边想”。

---

## 4. 最推荐的推进路线

## 第一步：做“主 PRD 收敛”

### 目标

把前面所有关键讨论，收敛成一份**唯一主文档**。

这份 PRD 不需要写得很官样文章，  
但必须成为之后所有 UI / 实现的主依据。

### 这份 PRD 需要合并的核心结论

至少包括：

1. **页面一句话定位**
2. **单用户模式**
3. **三分区结构**
   - 入选区
   - 商品区
   - 自由素材区
4. **输入合同**
   - 封面：唯一
   - 商品名称：唯一
   - 文案：唯一
   - 音频：唯一
   - 视频：集合
5. **主题商品默认规则**
6. **用户手改后的脱钩规则**
7. **版本创建时机**
8. **结果确认流**
9. **P0 页面目标**

### 为什么这一步最重要

因为从这一步开始：

> **我们不再依赖“很多篇讨论文档的组合理解”，而是依赖一份主设计合同。**

---

## 第二步：做“整页 UI 蓝图”

### 目标

不是直接出高保真视觉稿，  
而是先做：

> **整页低保真线框 + 信息架构蓝图**

这一步要回答：

- 首屏先看到什么
- 三个区怎么排
- 每个区显示哪些字段
- 每个区有哪些关键动作
- CTA 放哪里

### 关键要求

这一步不是继续谈业务原则，  
而是把 PRD 翻译成页面结构。

也就是说：

> **PRD 解决“是什么”，UI 蓝图解决“怎么摆”。**

---

## 第三步：做“实现切片计划”

### 目标

把整页重构拆成可以执行的小步。

建议至少拆成：

### P0

- 三分区骨架
- 当前入选区重建
- 商品选择入口升级
- 商品默认/手改脱钩规则

### P1

- 商品区 richer picker
- 展开查看
- 自由素材区更完整联动

### P2

- 更强筛选
- 更强可视化预览
- 更完整结果确认/发布衔接

### 为什么必须切片

因为现在如果一次性重做整页，风险会很高。  
先把 P0 做对，比“试图一次完美”重要得多。

---

## 第四步：只先做 P0

这是最关键的执行纪律。

P0 的目标不是：

- 把所有问题一次解决

而是：

> **先让 CreativeDetail 的页面心智正确起来。**

也就是：

- 页面结构正确
- 字段关系正确
- 默认值/控制权正确
- 当前入选区正确

只要这几个点对了，后面的增强才值得做。

---

## 5. 所以现在你的最佳下一步，不是“PRD 还是 UI 二选一”

而是：

> **先用一份主 PRD 把之前所有讨论收敛，再基于它画整页低保真，然后再切 P0 实现。**

换句话说：

### 不是这样：

- 继续讨论更多细节
- 或直接重做 UI

### 而是这样：

- **收敛主文档**
- **做整页蓝图**
- **分阶段实现**

### 具体执行顺序

## 8. 最终结论

你现在不需要纠结：

- 是不是该马上重做 UI
- 是不是该马上开始实现

因为最佳路径已经很明确：

> **先收敛成一份主 PRD，再基于它出整页 UI 蓝图，然后切片实施。**

一句话总结：

> **现在最该做的是“收敛主线”，不是“继续发散”，也不是“直接开工”。**
