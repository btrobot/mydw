# 远端 Day 4.7 实现记录（统一 route contract test helper）

## 目标

在**不改运行时合同**的前提下，收敛远端后端 route contract 测试中的重复样板，降低后续补回归入口时的复制成本。

## 本次落地

### 1. 抽出共享测试 helper

新增：

- `remote/remote-backend/tests/support/__init__.py`
- `remote/remote-backend/tests/support/route_contract.py`

统一承载以下测试辅助能力：

- `ErrorContractCase`：统一错误合同断言入参
- `RecordingRouteService`：统一记录 fake service 调用
- `bearer_headers()`：统一 bearer header 构造
- `expected_access_token()`：统一 401 场景下的 token 期望
- `assert_json_model_response()`：统一 model json 响应断言
- `assert_error_response()`：统一错误响应断言
- `assert_single_service_call()`：统一服务调用断言

其中 `RecordingRouteService.handle()` 允许“仅记录调用、不预置成功响应”的 destructive fake service 继续复用，避免 helper 抽取后反而破坏既有成功路径测试。

### 2. 迁移已补的 route contract 测试到共享 helper

已接入：

- `remote/remote-backend/tests/api/test_admin_destructive_api.py`
- `remote/remote-backend/tests/api/test_admin_read_api.py`
- `remote/remote-backend/tests/api/test_auth_refresh_api.py`
- `remote/remote-backend/tests/api/test_auth_session_api.py`
- `remote/remote-backend/tests/api/test_self_service_api.py`

## 风险判断

这是一次**纯测试侧整理**：

- 不改任何运行时代码
- 不改 API schema
- 不改错误语义
- 不改限流、鉴权、路由行为

最主要风险是 helper 抽取后把原本分散在各测试文件里的细节“抽平”，导致某些调用断言变弱。当前通过保留逐路由的参数断言与 targeted/full pytest 回归来兜底。

## 结论

Day 4.7 可以作为一个独立、低风险的“测试样板收敛”切片提交。后续若继续补新的 route contract 回归入口，应优先复用 `tests/support/route_contract.py`，避免再次复制 fake service / error assertion / bearer header 处理逻辑。
