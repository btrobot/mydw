## 取证结论

这次用户看到的左侧导航问题，实际发生在 `remote-admin` 的 **legacy 静态壳**，不是并行推进中的 React 壳。

- 本地启动脚本默认打开的是：
  - `index.html?apiBase=http://127.0.0.1:8100`
- 这条入口对应的是：
  - `src/main.ts`
  - `src/app/App.ts`
- 我之前修的是 React 壳：
  - `src/app/AppShell.tsx`
  - `src/app/adminTheme.ts`

所以之前那一刀“没效果”，不是修复没生效，而是 **用户实际打开的不是那套 UI**。

## 真正的根因

用户的直觉“导航栏和内容栏一样高度导致被拉长”是对的，但实现层面的根因更具体：

### 1. 第一层拉伸：`.sidebar`

`App.ts` 里旧壳的 sidebar 是：

```css
.sidebar { display: grid; gap: 20px; }
```

而外层 shell 又是：

```css
.shell { display: grid; grid-template-columns: 220px 1fr; min-height: 100vh; }
```

由于右侧主内容比 100vh 更高，左侧 `.sidebar` 会跟着整列一起变高。

### 2. 第二层拉伸：`.nav`

旧壳的导航本身又是：

```css
.nav { display: grid; gap: 8px; }
```

结果：

- `.sidebar` 里的 `brand` 和 `nav` 两块先被拉高
- `.nav` 里的每个链接又继续被 grid 拉伸
- 最终每个菜单项都变成大块，高度大约一百多像素

这就是为什么视觉上会觉得：

- 品牌区和菜单之间空得离谱
- 菜单项彼此之间像被“稀释”开了
- 整个左侧像被内容区高度拖着撑长

## 最小风险修复

这次不改结构、不切入口，只修 legacy 壳当前用户实际看到的 CSS：

- `.sidebar`：`grid -> flex column`
- `.nav`：`grid -> flex column`
- sidebar gap：`20px -> 16px`

这样能保证：

- 左侧整列仍可跟随页面高度
- 但品牌区和菜单区不再被 grid 自动拉伸
- 菜单项恢复为内容自然高度
- 风险保持在纯样式层

## 后续判断

如果后续确认要把 React 壳正式转正，再单独处理启动入口切换即可；这次先解决“用户当前实际打开的页面不好看”这个问题。
