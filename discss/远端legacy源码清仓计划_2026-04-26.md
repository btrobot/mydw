# 远端 legacy 源码清仓计划（2026-04-26）

## 本 slice 目标
- 把 `remote/remote-admin/src/main.ts` 与 `remote/remote-admin/src/app/App.ts` 从运行时与测试链路里安全移除。
- 不改当前 React 单入口运行时合同：`/index.html` 仍然是唯一入口。
- 先补 React 时代的最小纯函数 / contract 覆盖，再删旧文件，避免“删掉旧壳 = 失去最基本回归锚点”。

## 当前盘点
- 运行时入口已经切到 `src/react-main.tsx`，`src/main.ts` 事实上已不再被 `index.html` 引用。
- `src/app/App.ts` 只剩 legacy HTML 字符串渲染与旧路由守卫/纯函数集合。
- 当前唯一还直接依赖 legacy 构建产物的是：
  - `remote/remote-admin/tests/app.test.mjs`
  - 它从 `../dist/app/App.js` 导入旧守卫、旧 query builder、旧 renderApp。

## 最小风险改造策略
1. **先抽 React 时代仍然需要的纯辅助逻辑**
   - 新建 route helper（默认跳转、受保护路由跳转、页面标题）。
   - 让 React `router.tsx` / `AppShell.tsx` 直接复用同一份 helper。
2. **把测试从 legacy renderApp 改成 React-era contract test**
   - 保留：
     - 登录/动作错误映射
     - query builder 顺序与 UTC 边界
     - step-up header / verify 请求合同
     - offset pagination helper
     - route guard / title helper
   - 删除：
     - 依赖 `renderApp()` 的整页 HTML 字符串快照式断言
3. **确认无引用后删除旧文件**
   - 删除 `src/main.ts`
   - 删除 `src/app/App.ts`

## 验收标准
- `rg` 搜索结果中不再有业务代码/测试依赖 `src/main.ts`、`src/app/App.ts`、`dist/app/App.js`
- `npm --prefix remote/remote-admin run build`
- `npm --prefix remote/remote-admin run build:react`
- `npm --prefix remote/remote-admin test`
- `npm --prefix remote/remote-admin run test:react-step-up`

## 风险提醒
- 这次不顺手清 React 页面文案里的历史 Day 标注；先聚焦 legacy 代码退场，避免范围膨胀。
- 不在本 slice 顺手改运行时、样式系统或 query key 结构。

## 实施结果
- 已新增 `src/routes/route-helpers.ts`，把根跳转、受保护路由跳转、页面标题映射抽成 React 时代的纯 helper。
- `router.tsx` / `AppShell.tsx` 已改为复用上述 helper，不再依赖 legacy 守卫逻辑。
- 旧测试 `tests/app.test.mjs` 已替换为 `tests/react-contract.test.mjs`：
  - 保留 query / error mapping / pagination / step-up request contract 覆盖
  - 新增 route helper 覆盖
  - 删除 legacy `renderApp()` HTML 字符串断言
- 已删除：
  - `remote/remote-admin/src/main.ts`
  - `remote/remote-admin/src/app/App.ts`

## 本次验证
- `npm --prefix remote/remote-admin run build`
- `npm --prefix remote/remote-admin run build:react`
- `npm --prefix remote/remote-admin test`
- `npm --prefix remote/remote-admin run test:react-step-up`

## 当前结论
- remote-admin 已不再依赖 legacy 单页字符串渲染入口。
- 剩余风险主要不是 legacy 入口，而是 React 壳本身仍有体积偏大告警（既有问题，未在本 slice 处理）。
