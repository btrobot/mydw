# 远端 Day 5.1 实现记录（Alembic baseline 接入）

## 目标

开始落地 Slice 10，但严格停在 **Day 5.1：建立 Alembic baseline，不切运行时入口**。

本次目标是：

- 给 `remote-backend` 接入 Alembic 基础设施
- 固化一份**当前 schema 的 baseline revision**
- 让空库可以通过 Alembic 建出当前 schema
- 让已经由 legacy runner 初始化过的库可以被 Alembic **stamp 接管**
- 不在这一刀里切换现有 bootstrap/runtime 入口

## 代码事实基线

- 当前仍使用自定义 runner：`remote/remote-backend/app/migrations/runner.py`
- 当前 schema 真相来自 ORM：`remote/remote-backend/app/models.py`
- 当前 README 的本地运行方式仍是：
  - `python -c "from app.migrations.runner import upgrade; upgrade()"`
- bootstrap 脚本仍直接依赖 legacy runner：
  - `remote/remote-backend/scripts/bootstrap_admin.py`

这意味着 Day 5.1 的正确切法不是“直接替换主入口”，而是先建立 Alembic 基线与验证路径。

## 本次落地

### 1. 声明 Alembic 依赖

更新：

- `remote/remote-backend/pyproject.toml`

新增：

- `alembic>=1.13.0`

### 2. 建立 Alembic 基础配置

新增：

- `remote/remote-backend/alembic.ini`
- `remote/remote-backend/migrations/env.py`
- `remote/remote-backend/migrations/script.py.mako`

设计要点：

- Alembic script location 使用顶层 `migrations/`
- `env.py` 会主动把 backend 项目根加入 `sys.path`
- 默认数据库 URL 走 `app.core.config.get_settings().DATABASE_URL`
- 测试/脚本可通过 Alembic config 覆盖 `sqlalchemy.url`

这样既能兼容当前项目结构，也能在测试里对临时 sqlite 库做独立验证。

### 3. 固化当前 schema 的 baseline revision

新增：

- `remote/remote-backend/migrations/versions/20260425_0001_alembic_baseline.py`

本次 baseline 没有继续引用运行时 `Base.metadata.create_all()`，而是把当前 schema **冻结为显式 Alembic DDL**，覆盖：

- `users`
- `user_credentials`
- `licenses`
- `user_entitlements`
- `devices`
- `user_devices`
- `sessions`
- `refresh_tokens`
- `admin_users`
- `admin_sessions`
- `audit_logs`

这样后续即使 ORM 模型继续演进，baseline revision 仍然保持历史不可变。

### 4. 补 migration baseline 自动化回归

新增：

- `remote/remote-backend/tests/migrations/test_alembic_baseline.py`

覆盖三条最小关键路径：

1. **空库 upgrade head**
   - 能创建当前 schema
   - current revision 为 `20260425_0001`

2. **downgrade base**
   - revision 状态清空
   - 业务表被回滚

3. **legacy runner 已初始化的库 -> Alembic stamp**
   - 先用旧 runner 建表
   - 再用 `alembic stamp 20260425_0001`
   - 确认 Alembic 能接管现有库

这第三条很关键，因为它说明 Day 5.1 不要求强行重跑 baseline DDL，也为 Day 5.2 的主路径切换留出了平滑接入点。

### 5. 更新 Day 5.1 阶段文档

更新：

- `remote/remote-backend/README.md`

明确区分：

- **空库**：`alembic upgrade head`
- **已有 legacy runner 库**：`alembic stamp 20260425_0001`

并保留说明：

- 当前 runtime/bootstrap 仍可能继续调用 `app.migrations.runner`
- 主入口切换属于后续 Slice 10 步骤

## 为什么这是最小风险 Day 5.1

因为这一步只做三件事：

1. **建立 Alembic 能力**
2. **冻结 baseline revision**
3. **验证空库/旧库都能纳入 Alembic 视角**

而没有做这些高风险动作：

- 没有替换生产/开发启动主路径
- 没有修改 bootstrap_admin 的运行方式
- 没有删除旧 runner
- 没有把测试初始化全部切到 Alembic

所以它是典型的“先搭桥，再切流量”切片。
