# 远端页面迁移验收清单

日期：2026-04-27

## 1. 用法

每迁完一个页面，不要靠“感觉差不多了”判断完成，而是用这份清单逐项过。

规则：

1. 页面主路径可走通
2. loading / empty / error 至少覆盖主要状态
3. 角色差异没有破坏
4. destructive action 没有误开放

---

## 2. login 页

- [ ] 未登录访问受保护页面会被重定向到 login
- [ ] 输入正确账号密码可以登录
- [ ] 登录成功后进入 dashboard
- [ ] 错误账号密码显示稳定错误文案
- [ ] forbidden 账号显示稳定错误文案
- [ ] too_many_requests 显示稳定错误文案
- [ ] 本地 access token 存储与恢复正常

---

## 3. dashboard 页

- [ ] 已登录用户可进入 dashboard
- [ ] metrics 能正常加载
- [ ] loading 态可见
- [ ] error 态可见
- [ ] retry 可触发重新加载
- [ ] readonly 角色能正常查看 dashboard

---

## 4. users 页

- [ ] users 列表能加载
- [ ] query / status / license_status 筛选可用
- [ ] 列表切换详情可用
- [ ] update user 主路径可用
- [ ] revoke 可用
- [ ] restore 可用
- [ ] readonly 角色不可执行 destructive action
- [ ] 列表 loading / empty / error 主态正常
- [ ] 详情 loading / error 主态正常

---

## 5. devices 页

- [ ] devices 列表能加载
- [ ] query / status / userId 筛选可用
- [ ] 列表切换详情可用
- [ ] unbind 可用
- [ ] disable 可用
- [ ] rebind 可用
- [ ] readonly 角色不可执行 destructive action
- [ ] 列表 loading / empty / error 主态正常
- [ ] 详情 loading / error 主态正常

---

## 6. sessions 页

- [ ] sessions 列表能加载
- [ ] query / authState / userId / deviceId 筛选可用
- [ ] 选择 session 可用
- [ ] revoke session 可用
- [ ] readonly 角色不可执行 destructive action
- [ ] 列表 loading / empty / error 主态正常

---

## 7. audit 页

- [ ] audit 列表能加载
- [ ] eventType / actorId / target* / time range 筛选可用
- [ ] limit / offset 翻页可用
- [ ] 选择 audit 记录查看详情可用
- [ ] loading / empty / error 主态正常

---

## 8. 首轮迁移的最低通过线

在首轮实施里，不要求所有页面一次性迁完，但要求：

### 第一批最低通过线

- [ ] login 通过
- [ ] dashboard 通过
- [ ] users 至少完成列表与角色态基础能力

### 未达到通过线时的处理

- 不继续推进 devices / sessions / audit
- 不插入分页统一 / Alembic / 安全增强等旁支任务
- 优先先把主路径与测试保护网补齐
