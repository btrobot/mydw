# 远端 Day 5.3 实现记录（退役旧 runner，完成 Alembic 标准化收口）

## 目标

完成 Slice 10 最后一刀，把迁移路径从“新旧并存”收口到 **Alembic 成为唯一标准入口**。

本次要求：

- 删除旧 runner 主路径
- 清理剩余调用面与文档引用
- 保留“接管 pre-Alembic 旧库”的最小兼容能力
- 让后续 schema 变更路径只剩 Alembic revision

## 本次改动

### 1. 删除 legacy runner 代码

已删除：

- `remote/remote-backend/app/migrations/runner.py`
- `remote/remote-backend/app/migrations/versions/migration_0001_initial_auth_core.py`
- `remote/remote-backend/app/migrations/versions/migration_0002_add_access_token_hash.py`
- `remote/remote-backend/app/migrations/versions/migration_0003_add_admin_auth.py`
- `remote/remote-backend/app/migrations/versions/__init__.py`

这一步的意义是把“旧路径还能不能继续偷偷用”直接收口掉，避免后续 schema 变更继续分叉。

### 2. 把兼容逻辑从“legacy runner”语义改成“pre-Alembic schema 接管”

修改：

- `remote/remote-backend/app/migrations/alembic.py`
- `remote/remote-backend/scripts/migrate.py`

变化：

- 保留 `ensure_database_on_head()`
- 但内部不再把它建模成“继续依赖旧 runner”
- 而是建模为：**如果数据库已有业务表但还没有 Alembic revision，就先 stamp baseline，再 upgrade head**

也就是说，兼容的是“旧状态”，不是“旧执行器”。

### 3. 更新剩余调用面

修改：

- `remote/remote-shared/scripts/compat-harness/validate_phase1_gate.py`

本地兼容门禁的 Local harness 初始化不再 import 旧 runner，而是直接调用：

- `app.migrations.alembic.ensure_database_on_head()`

这样连 shared tooling 也回到同一条迁移路径。

### 4. 更新文档与约束

修改：

- `remote/remote-backend/README.md`
- `remote/README.md`
- `remote/remote-shared/docs/staging-deploy-checklist.md`

文档统一改为：

- 本地 / staging 初始化走 `python scripts/migrate.py ensure-head`
- 新增 schema 变更必须通过 Alembic revision
- 不再允许通过旧 runner 或手工维护 `schema_migrations` 演进数据库

### 5. 调整测试，保留旧库接管回归

修改：

- `remote/remote-backend/tests/migrations/test_alembic_baseline.py`
- `remote/remote-backend/tests/migrations/test_alembic_entrypoints.py`
- 新增 `remote/remote-backend/tests/migrations/support.py`

由于旧 runner 已删除，测试不再 import 它来造“旧库”。

新的做法是：

1. 先用 Alembic baseline 建出现有 schema
2. 手工移除 `alembic_version`
3. 补出 `schema_migrations`

从而模拟 **pre-Alembic 已存在数据库**，继续验证：

- `stamp baseline`
- `ensure_database_on_head()`

都还能平滑接管旧库。

## 结果判断

Day 5.3 完成后，Slice 10 可以视为收口完成：

- 运行路径：Alembic
- bootstrap 路径：Alembic
- 测试初始化路径：Alembic
- shared tooling 初始化路径：Alembic
- 文档默认路径：Alembic

剩下保留的只有“旧库接管能力”，不再保留“旧 runner 继续可用”的主路径。
