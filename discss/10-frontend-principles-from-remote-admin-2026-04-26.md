# 从 Remote Admin 提炼出 10 条前端可直接照搬的原则

## 1. 先做壳层，再做页面

不要一上来就写 `UsersPage`、`OrdersPage`。

先做：

- `AppShell`
- `Sidebar`
- `Header`
- `Content`

因为真正决定“像不像一个产品”的，首先不是页面内容，而是整个系统有没有统一骨架。

**可直接照搬：**

- 路由先搭 layout route
- 所有登录后页面统一挂在 `AppShell` 下

---

## 2. 不要按页面开发，要按页面模式开发

不要把每个页面都当成一个新世界。

后台前端常见页面其实就几种：

- Dashboard
- 列表 + 详情（master-detail）
- 审计/日志/排查页

**可直接照搬：**

先定义 3 套模板：

1. Dashboard 模板
2. CRUD 模板
3. Audit/Logs 模板

以后新页面优先“套模板”，不是“重新设计”。

---

## 3. 先定 token，再写样式

不要边写页面边临时拍脑袋填颜色和间距。

先统一：

- 主背景色
- 侧栏背景色
- 品牌色
- 边框色
- 圆角
- 间距
- 控件高度

**可直接照搬：**

先建一个：

- `theme.ts`
- 或 `tokens.ts`

然后页面只能用 token，不要乱写硬编码样式。

---

## 4. 所有列表页都统一成一套数据流

列表页最怕每页各写一套。

统一数据流：

1. `draftFilters`
2. `appliedFilters`
3. `listQuery`
4. `detailQuery`
5. `selectedId`
6. `mutation`
7. `actionFeedback`
8. `actionError`

**可直接照搬：**

以后所有：

- users
- devices
- sessions
- orders
- members

都按同一套路写。

---

## 5. loading / empty / error 不是补丁，是标配

不要只做“正常显示”的页面。

每个页面都必须先想清楚：

- 首屏加载显示什么
- 没数据显示什么
- 出错显示什么
- 重试怎么做

**可直接照搬：**

统一抽：

- `LoadingState`
- `EmptyState`
- `ErrorState`

不要每页自己拼。

---

## 6. 权限不是逻辑判断，是界面语言

不要只写：

- `disabled={true}`

用户需要知道：

- 为什么不能点
- 当前角色是什么
- 谁才有权限

**可直接照搬：**

前端统一补齐这 3 个东西：

1. `RoleBadge`
2. `ReadonlyNotice`
3. 禁用原因文案

这样页面会立刻更像企业产品。

---

## 7. 危险操作必须单独成区，不要混在普通操作里

后台前端专业不专业，很大程度看危险操作怎么处理。

像：

- revoke
- disable
- delete
- restore

不要和普通“保存”“刷新”混在一起。

**可直接照搬：**

统一抽一个：

- `DangerZone`

里面固定包含：

- 风险说明
- 按钮
- confirm
- in-flight 状态
- 成功/失败反馈

---

## 8. 前端状态一定要分层，不要混着管

最容易把项目搞乱的，就是把所有状态塞一起。

至少分 3 层：

1. **服务端状态**
   - 列表
   - 详情
   - metrics

2. **本地界面状态**
   - modal 开关
   - 选中项
   - tab
   - filter draft

3. **全局会话状态**
   - 当前用户
   - token
   - role
   - session 过期

**可直接照搬：**

- React Query 管服务端状态
- component state 管本地交互
- Auth context/store 管全局会话

---

## 9. 文案也是前端体验的一部分

很多人只盯组件，不盯文案。

但后台页面“专业不专业”，很多时候就差在这些地方：

- 副标题会不会解释清楚
- 空态是不是有指导意义
- 错误态是不是可恢复
- 危险操作是不是把后果说清楚

**可直接照搬：**

前端组件不要只收：

- `title`

最好还支持：

- `subtitle`
- `description`
- `hint`
- `warning`
- `reason`

---

## 10. 先做“系统一致”，再做“单页精致”

很多前端最大的问题不是某页不精致，而是：

- 这一页像 A 产品
- 那一页像 B 产品
- 第三页像 demo

Remote Admin 最值得抄的，不是某个卡片，而是：

> 整个系统先统一，再局部精修。

**可直接照搬：**

你的开发顺序应该是：

1. 统一壳层
2. 统一 token
3. 统一状态组件
4. 统一页面模板
5. 再逐页打磨

不要反过来。

---

# 最后给你一个超短版

如果只记 10 句话，就记这 10 句：

1. 先做壳层，不先做散页  
2. 按页面模式开发，不按页面名字开发  
3. 先定 token，再写样式  
4. 列表页统一数据流  
5. loading/empty/error 必须组件化  
6. 权限要显式表达，不要只 disabled  
7. 危险操作单独成区  
8. 服务端状态、本地状态、会话状态必须分层  
9. 文案也是前端设计的一部分  
10. 先保证全局一致，再做单页精修

---

# 一句话总结

> **Remote Admin 最值得前端照搬的，不是某个 UI 样式，而是“先做系统，再做页面；先做规则，再做细节”的方法。**
