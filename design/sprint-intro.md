● Sprint Plan Skill 使用指南

  触发方式
                                                                                                                                     /sprint-plan          # 默认创建新 Sprint
  /sprint-plan new      # 创建新 Sprint                                                                                              /sprint-plan status   # 查看 Sprint 状态
  /sprint-plan update   # 更新 Sprint 进度

  ---
  使用示例

  示例 1：创建新 Sprint

  /sprint-plan new

  输出示例：

  ## Sprint 规划

  请提供以下信息：

  1. Sprint 名称：[例如：AI 剪辑功能]
  2. Sprint 周期：YYYY-MM-DD ~ YYYY-MM-DD（2周）
  3. Sprint 目标：
     - [目标1]
     - [目标2]
  4. 任务列表（每个任务包含）：
     - 任务描述
     - 负责人（Agent）
     - 估计时间（人天）
     - 依赖关系

  示例 2：实际创建 Sprint

  假设用户回答：
  - 名称: Sprint 6 - 用户认证模块
  - 周期: 2026-04-07 ~ 2026-04-18
  - 目标:
    - 完成用户登录功能
    - 完成 Cookie 加密存储
  - 任务:
    - 登录 API 开发
    - Cookie 加密服务
    - 前端登录界面
    - 安全测试

  创建的文件：

  # production/sprints/sprint-6.md

  ## Sprint 6 - 用户认证模块
  **周期**: 2026-04-07 ~ 2026-04-18 (2 周)

  ### 目标
  - 完成用户登录功能
  - 完成 Cookie 加密存储

  ### 任务
  | ID | 任务 | 负责人 | 估计 | 依赖 | 状态 |
  |----|------|--------|------|------|------|
  | SP6-01 | 登录 API 开发 | backend-lead | 2d | - | [ ] |
  | SP6-02 | Cookie 加密服务 | backend-lead | 1d | SP6-01 | [ ] |
  | SP6-03 | 前端登录界面 | frontend-lead | 2d | SP6-01 | [ ] |
  | SP6-04 | 安全测试 | security-expert | 1d | SP6-02 | [ ] |

  ### 容量
  - 总估计: 6 人天
  - 缓冲: 20%
  - 实际可用: 8 人天 ✓

  ### 风险
  | 风险 | 概率 | 影响 | 缓解 |
  |------|------|------|------|
  | 加密实现复杂度 | 中 | 中 | 参考现有 crypto.py |

  ### 状态
  - 开始日期: 2026-04-07
  - 结束日期: 2026-04-18
  - 进度: 0%

  示例 3：查看 Sprint 状态

  /sprint-plan status

  输出示例：

  ## Sprint 6 - 用户认证模块

  **进度**: 2/6 任务完成 (33%)

  ### 任务状态
  | ID | 任务 | 状态 |
  |----|------|------|
  | SP6-01 | 登录 API 开发 | ✅ 完成 |
  | SP6-02 | Cookie 加密服务 | 🔄 进行中 |
  | SP6-03 | 前端登录界面 | 🔄 进行中 |
  | SP6-04 | 安全测试 | ⏳ 待开始 |

  ### 剩余时间
  5 天（进度正常）

  工作流程

  ┌─────────────────────────────────────────────────────────────┐
  │                    Sprint 生命周期                           │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │  1. 创建 Sprint                                              │
  │     /sprint-plan new                                        │
  │         ↓                                                   │
  │  2. 任务分解                                                │
  │     /task-breakdown                                        │
  │         ↓                                                   │
  │  3. 执行开发                                                │
  │     → backend-lead (API)                                   │
  │     → frontend-lead (UI)                                    │
  │     → security-expert (安全)                                │
  │         ↓                                                   │
  │  4. 代码审查                                                │
  │     /code-review                                           │
  │         ↓                                                   │
  │  5. 更新状态                                                │
  │     /sprint-plan update                                    │
  │         ↓                                                   │
  │  6. 回顾总结                                                │
  │     /sprint-plan complete                                   │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  关键原则

  ┌────────────┬─────────────────────────┐
  │    原则    │          说明           │
  ├────────────┼─────────────────────────┤
  │ 任务大小   │ 每个任务 1-3 天完成     │
  ├────────────┼─────────────────────────┤

  ├────────────┼─────────────────────────┤

  ├────────────┼─────────────────────────┤

  ├────────────┼─────────────────────────┤
