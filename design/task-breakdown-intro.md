● Task Breakdown Skill 使用指南

  触发方式
                                                                                                                                     /task-breakdown                    # 交互式分解
  /task-breakdown AI 剪辑            # 直接分解指定功能                                                                              /task-breakdown 用户登录            # 直接分解指定功能

  ---
  任务分解层次

  Epic (大功能)
   └── Feature (子功能)
        └── User Story (用户故事)
             └── Task (开发任务) ← 这里分解的目标

  ---
  使用示例

  示例 1：交互式分解

  /task-breakdown

  输出：

  ## 任务分解

  请描述要分解的功能：
  1. 功能名称：
  2. 核心用户故事：
  3. 主要用户流程：

  ---
  示例 2：分解"视频上传"功能

  输入：/task-breakdown 视频上传

  输出：

  # 视频上传功能任务分解

  ## 功能概述
  用户选择本地视频文件，上传到服务器进行处理。

  ## 任务分解

  ### 后端任务

  #### BE-UPLOAD-01: 文件上传 API

  **描述**: 实现视频文件上传接口

  **验收标准**:
  - [ ] POST /api/upload
  - [ ] 支持 mp4, avi, mov 格式
  - [ ] 最大文件 2GB
  - [ ] 返回文件 ID

  **估计**: 1d
  **负责人**: backend-lead
  **类型**: backend

  ---

  #### BE-UPLOAD-02: 文件存储服务

  **描述**: 管理上传文件的存储和清理

  **验收标准**:
  - [ ] 存储到指定目录
  - [ ] 生成唯一文件名
  - [ ] 清理过期文件

  **依赖**: BE-UPLOAD-01
  **估计**: 0.5d
  **负责人**: backend-lead
  **类型**: backend

  ---

  ### 前端任务

  #### FE-UPLOAD-01: 上传组件

  **描述**: 视频上传界面组件

  **验收标准**:
  - [ ] 拖拽上传
  - [ ] 进度显示
  - [ ] 错误提示

  **依赖**: BE-UPLOAD-01
  **估计**: 1d
  **负责人**: frontend-lead
  **类型**: frontend

  ---

  ### 集成任务

  #### INT-UPLOAD-01: API 类型对齐

  **描述**: 前后端 API 类型定义

  **依赖**: BE-UPLOAD-01, FE-UPLOAD-01
  **估计**: 0.5d
  **负责人**: frontend-lead + backend-lead
  **类型**: both

  ---

  ## 任务汇总

  | ID | 任务 | 负责人 | 估计 | 依赖 |
  |----|------|--------|------|------|
  | BE-UPLOAD-01 | 文件上传 API | backend-lead | 1d | - |
  | BE-UPLOAD-02 | 文件存储服务 | backend-lead | 0.5d | BE-01 |
  | FE-UPLOAD-01 | 上传组件 | frontend-lead | 1d | BE-01 |
  | INT-UPLOAD-01 | API 类型对齐 | Lead x2 | 0.5d | BE-01, FE-01 |

  **总计**: 3d | **缓冲后**: 3.6d (~4d)

  任务模板

  每个分解出的任务包含：

  ### [ID] 任务名称

  **描述**: [具体做什么]

  **验收标准**:
  - [ ] 标准 1
  - [ ] 标准 2

  **估计**: Xd

  **负责人**: [Agent]

  **依赖**:
  - [依赖任务 ID]

  **类型**: frontend | backend | both

  **测试需求**:
  - [ ] 单元测试
  - [ ] 集成测试

  工作流程

  ┌─────────────────────────────────────────────────────────────┐
  │                   任务分解流程                              │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │  1. 输入功能                                                │
  │     /task-breakdown 视频上传                                │
  │         ↓                                                   │
  │  2. 理解功能                                                │
  │     - 阅读需求文档                                          │
  │     - 识别核心用例                                          │
  │         ↓                                                   │
  │  3. 识别参与者                                              │
  │     - 前端：frontend-lead                                  │
  │     - 后端：backend-lead                                    │
  │     - 自动化：automation-developer                          │
  │         ↓                                                   │
  │  4. 分解任务                                                │
  │     - 按前端/后端/集成/测试分类                              │
  │     - 定义验收标准                                          │
  │     - 估计工时                                                │
  │     - 标记依赖关系                                          │
  │         ↓                                                   │
  │  5. 汇总输出                                                │
  │     - 任务汇总表                                            │
  │     - 总工时估算                                            │
  │     - 建议的 Sprint 周期                                    │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
