# Epic 7 Version Inventory

> PR3 checked-in artifact  
> 目的：明确用户可见版本元信息的 canonical source 与同步范围，并说明哪些 package-manager metadata 不属于用户可见 canonical source。

## Canonical version source

- `backend/core/config.py`
- constant: `APP_VERSION`

当前 canonical version：

- `0.2.0`

## User-visible version surfaces that must stay in sync

| Surface | File | Sync rule |
|---|---|---|
| README displayed version | `README.md` | synced by `npm run version:sync` from `APP_VERSION` |
| FastAPI app version | `backend/main.py` | must read `settings.APP_VERSION` |
| backend root metadata | `backend/main.py` | must return `settings.APP_VERSION` |
| frontend package version | `frontend/package.json` | synced by `npm run version:sync`; consumed as Electron app version |
| Electron about/version string | `frontend/electron/main.ts` | must display `app.getVersion()` |
| tracked generated Electron runtime JS mirror | `frontend/electron/main.js` | must stay aligned with `frontend/electron/main.ts`; generated mirror, not handwritten source |
| CLI package version | `frontend/scripts/cli/package.json` | synced by `npm run version:sync` |
| CLI `--version` output | `frontend/scripts/cli/src/version.ts` → `frontend/scripts/cli/src/index.ts` | generated by `npm run version:sync`; not hand-maintained in `index.ts` |
| tracked generated OpenAPI snapshot `info.version` | `frontend/openapi.local.json` | regenerated/exported from backend version; snapshot is not the canonical version source |

## Standard sync/check commands

从 `frontend/` 运行：

```bash
npm run version:sync
npm run version:check
```

- `version:sync`：从 `backend/core/config.py` 读取 `APP_VERSION`，同步 `README.md`、frontend/CLI package metadata、CLI generated version module，以及 lockfile root metadata；随后刷新 `openapi.local.json`
- `version:check`：校验上述 version-bearing surfaces 与 canonical version 一致，避免 release/review 时再靠人工 grep 猜测

## Generated mirrors inside version surfaces

下列文件会携带版本信息，但它们本身不是 canonical source：

- `frontend/electron/main.js`：由 `frontend/electron/main.ts` 编译得到的 generated JS mirror
- `frontend/openapi.local.json`：由 backend OpenAPI 导出的 generated snapshot

review 这些文件时，应先确认对应 source surface 是否改变，再看 regenerate 后 diff 是否可预测。

## Explicit non-canonical metadata

这些不作为“用户可见 canonical source”管理：

- `frontend/package-lock.json`
- `frontend/bun.lock`
- `frontend/scripts/cli/package-lock.json`
- third-party dependency versions inside lockfiles

原因：

- 它们记录的是包管理解析结果
- 不是当前产品/构建面对用户展示的版本答案
- 虽然 `version:sync` 会尽量同步 lockfile root metadata 以减少误导，但 lockfile 仍不应被当作 canonical version source

## Grep checklist

推荐检查命令：

```bash
cd frontend && npm run version:check
```

Pass condition:

- 所有用户可见版本 surfaces 收口到 `APP_VERSION`
- 仍残留的版本号若只存在于 lockfile / third-party metadata，必须被明确归类为 non-canonical metadata
