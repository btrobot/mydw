# 得物掘金工具 - 统一需求说明书

## 冲突摘要

本节汇总所有来源间的不一致之处，详细说明见各需求条目的冲突标记。

| 编号 | 冲突点 | 涉及来源 | 说明 |
|------|--------|----------|------|
| C1 | AI 剪辑输入数量 | init-req vs chat-req vs user-guide | init-req 未明确数量；chat-req 要求 2 个视频合成 1 个；user-guide 要求 2 个或以上 |
| C2 | 文案来源方式 | init-req vs chat-req vs user-guide | init-req 仅提"添加文案"（手动）；chat-req 要求 AI 生成；user-guide 提供"原文案"和"自动文案"两种模式 |
| C3 | 任务分配模式 | init-req vs chat-req | init-req 任务关联单个账号；chat-req 要求多账号轮询自动分配 |
| C4 | 流程自动化程度 | init-req vs chat-req | init-req 每步手动触发；chat-req 要求全自动流程（素材->分配->发布） |
| C5 | 发布模式 | init-req vs user-guide | init-req 提"乱序发布"；user-guide 提"顺序发布"和"直接发布"，未提乱序 |
| C6 | 素材导入方式 | init-req vs user-guide | init-req 为手动添加（添加视频/文案/封面等）；user-guide 增加了"扫描素材"自动导入 |
| C7 | 话题管理方式 | init-req vs user-guide | init-req 仅提"添加话题"；user-guide 描述了话题搜索、热度排序、批量设置的完整流程 |

## 一、账号管理

### 1.1 多账号管理

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |

- 支持添加、编辑、删除多个得物账号
- 账号信息包含：账号 ID、账号名称
- user-guide 补充：支持标签和备注字段

### 1.2 账号登录（连接）

| 属性 | 值 |
|------|-----|
| 来源 | user-guide |
| 优先级 | P0 |

- 通过内置浏览器窗口完成得物平台登录
- 登录方式：手机号 + 短信验证码
- 登录成功后保存 Session（Cookie），状态显示为"已连接"

### 1.3 连接状态检查

| 属性 | 值 |
|------|-----|
| 来源 | user-guide |
| 优先级 | P0 |

- 单账号"检查连接"：验证 Session 是否有效
- 批量检查连接：一次性检查所有账号状态
- Session 过期后提示"重新登录"

### 1.4 账号预览

| 属性 | 值 |
|------|-----|
| 来源 | user-guide |
| 优先级 | P1 |

- 打开可见浏览器窗口，展示账号在得物创作者平台的实际页面
- 关闭预览时可选择是否保存当前 Session

### 1.5 数据看板

| 属性 | 值 |
|------|-----|
| 来源 | init-req |
| 优先级 | P1 |

- 今日发布数、今日播放数、7 天发布总条数、总视频数
- 今日收益、总收益、今日总收益增长
- 列表总收益

## 二、素材管理

### 2.1 素材导入

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |
| **冲突** | **C6** |

- 支持的素材类型：视频、文案、封面、话题、音频
- init-req：手动添加各类素材，本地路径引用
- user-guide：增加"扫描素材"自动导入功能，也支持手动"导入素材"选择文件

> **冲突 C6 说明**: init-req 仅描述手动逐项添加；user-guide 增加了目录扫描自动导入。建议两种方式都支持。

### 2.2 商品信息管理

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |

- 添加商品：商品名称、分享文本（系统从分享文本提取 `dewu_url` 并立即解析素材）
- 刷新数据：同步商品信息
- 视频与商品绑定关联
- 文案/封面/话题与视频绑定关联

### 2.3 素材路径管理

| 属性 | 值 |
|------|-----|
| 来源 | init-req |
| 优先级 | P1 |

- 管理素材存储路径（ID、链接、本地路径）
- 本地路径引用配置

## 三、AI 剪辑合成

### 3.1 多视频合成

| 属性 | 值 |
|------|-----|
| 来源 | init-req, chat-req, user-guide |
| 优先级 | P0 |
| **冲突** | **C1** |

- 将多个源视频合成为一个新视频
- chat-req：明确要求 2 个视频合成 1 个新视频（当前只支持单视频输入）
- user-guide：选择 2 个或以上的源视频进行合成

> **冲突 C1 说明**: chat-req 限定为"2 合 1"；user-guide 为"2 个或以上"。建议支持 2 个及以上，满足两方需求。

### 3.2 合成参数配置

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |

- 合成时长设置（user-guide 建议 2-5 分钟）
- 合成超时分钟数设置
- 素材时长要求（user-guide 建议单个素材 30 秒以上）
- 合成引擎选择（init-req 提到豆包合成开关）

### 3.3 文案模式

| 属性 | 值 |
|------|-----|
| 来源 | init-req, chat-req, user-guide |
| 优先级 | P1 |
| **冲突** | **C2** |

- init-req：手动"添加文案"
- chat-req：要求文案由 AI 生成（当前从本地 .txt 文件读取）
- user-guide：提供两种模式 —"原文案"（使用源视频文案）和"自动文案"（系统生成）

> **冲突 C2 说明**: 三份来源对文案来源的描述不一致。建议支持三种模式：手动输入、原文案复用、AI 自动生成。

### 3.4 视频信息提取

| 属性 | 值 |
|------|-----|
| 来源 | user-guide |
| 优先级 | P1 |

- 合成完成后支持"提取"操作
- 自动提取视频标题和文案，用于后续发布

## 四、话题管理

### 4.1 话题设置

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P1 |
| **冲突** | **C7** |

### 5.1 任务列表

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |

- 任务字段：视频路径、视频文案、关联账号、发布状态、商品名称
- 操作：编辑、删除

| 属性 | 值 |
|------|-----|
| 来源 | chat-req |
| 优先级 | P0 |
| **冲突** | **C3** |

- chat-req：支持多账号轮询分配（例：10 个账号 12 个视频，轮询分配）
- init-req：任务关联单个账号（手动选择）

> **冲突 C3 说明**: init-req 为手动单账号关联；chat-req 要求自动轮询分配。建议同时支持手动指定和自动轮询分配两种模式。

### 5.3 任务初始化与清空

| 属性 | 值 |
|------|-----|

## 六、发布控制

### 6.1 发布策略配置

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |

- 发布间隔（分钟）
- 发布时间范围（user-guide 示例：9:00 - 22:00）
- 单账号每日发布条数（user-guide 示例：6 条）
- 发布日期设置（user-guide 补充）

### 6.2 发布模式

| 属性 | 值 |
|------|-----|
| 来源 | init-req, user-guide |
| 优先级 | P0 |
| **冲突** | **C5** |

- init-req：开始发布、乱序发布
- user-guide：顺序发布（按时间间隔依次）、直接发布（立即发布所有）

> **冲突 C5 说明**: init-req 提到"乱序发布"，user-guide 未提及，改为"顺序发布"和"直接发布"。建议支持三种模式：顺序发布、乱序发布、直接发布。

### 6.3 自动发布流程

| 属性 | 值 |
|------|-----|
| 来源 | user-guide |
| 优先级 | P0 |

发布时系统自动完成：
1. 打开对应账号的浏览器
2. 上传视频文件
3. 填入标题、文案、话题
4. 关联商品
5. 完成发布

### 6.4 全自动流程（端到端）

| 属性 | 值 |
|------|-----|
| 来源 | chat-req |
| 优先级 | P1 |
| **冲突** | **C4** |

- chat-req：要求全自动流程（素材 -> 分配 -> 发布），无需手动触发每步
- init-req：每步需手动操作

> **冲突 C4 说明**: chat-req 要求端到端自动化；init-req 设计为分步手动操作。建议先实现分步手动流程（P0），再叠加全自动模式（P1）。

### 6.5 数据刷新

| 属性 | 值 |
|------|-----|
| 来源 | init-req |
| 优先级 | P2 |

- 刷新数据按钮，同步最新发布状态

## 冲突决策建议

以下为 Tech Lead 对各冲突点的建议方案，待 Product Owner 确认：

| 冲突 | 建议决策 | 理由 |
|------|----------|------|
| C1 AI 剪辑输入数量 | 支持 2 个及以上源视频 | 向上兼容，满足 chat-req 的 2 合 1 场景，也支持 user-guide 的多视频场景 |
| C2 文案来源方式 | 支持三种模式：手动输入、原文案复用、AI 自动生成 | 覆盖所有来源的需求，用户按需选择 |
| C3 任务分配模式 | 同时支持手动指定和自动轮询分配 | 手动模式保留灵活性，自动轮询提升效率 |
| C4 流程自动化程度 | 分阶段实现：先分步手动（P0），再叠加全自动模式（P1） | 降低风险，渐进式交付 |
| C5 发布模式 | 支持三种：顺序发布、乱序发布、直接发布 | 合并两份来源的所有模式，给用户最大灵活性 |
| C6 素材导入方式 | 同时支持手动添加和目录扫描自动导入 | 两种方式互补，不冲突 |
| C7 话题管理方式 | 按 user-guide 的完整流程实现（搜索+热度+批量设置） | user-guide 描述更完整，init-req 的"添加话题"是其子集 |
