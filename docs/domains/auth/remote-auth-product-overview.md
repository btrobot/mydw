# Remote Auth 功能说明（面向产品 / 使用者）

## 1. 它解决什么问题

remote-auth 让这个工具不仅“能登录”，还能够：

- 确认当前机器是否被授权
- 在远程授权失效时及时限制危险操作
- 在短时离线时允许只读/受限使用
- 给用户明确的错误提示和恢复路径
- 让管理员查看并撤销本地授权会话

## 2. 用户能感受到的变化

### 登录更明确

- 输入错误账号密码时，会看到更友好的错误提示
- 网络问题时，会提示“授权服务不可达”，并给 Retry

### 启动更稳定

- 应用启动时会自动恢复本地授权状态
- 如果恢复失败，不会完全卡死，而是给出提示并允许进入可恢复界面

### 状态更清楚

当授权有问题时，用户会看到明确页面，例如：

- Authorization revoked
- Device mismatch
- Session expired
- Offline grace mode

### 离线更可控

- 短时网络问题下，系统进入 **grace mode**
- 可以继续看已有本地数据
- 但危险操作和新后台任务会被限制

## 3. 管理员能力

新增了 auth admin 页面，可以：

- 查看当前本地授权会话
- 区分当前会话和历史会话
- 主动 revoke 会话

适合：

- 排查授权异常
- 强制重新登录
- 清理遗留会话状态

## 4. 对业务的价值

这套 remote-auth 让产品从“本地工具”升级成了“**可控授权的本地工作台**”：

- **更安全**：设备不匹配、吊销、过期都能落地限制
- **更稳定**：离线/失败有 grace 和恢复路径
- **更可支持**：有状态 API、日志、metrics、admin 工具
- **更易用**：用户看到的错误更清晰，不再只是一句失败

## 5. 一句话总结

这已经不是“只会登录”的 auth，而是一套：

> **有本地会话真相、有状态机、有门禁、有日志指标、有错误 UX、可管理会话的 remote-auth 体系。**
