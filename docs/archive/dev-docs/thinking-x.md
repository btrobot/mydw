

❯ 思考了一下，这个视频合成与创作的过程。我们用素材编排了一个视频创作的任务，然后是视频合成（使用coze或者本地ffmpeg方案），然后是调 度上传（哪个任务通过哪个账号在什么时间上传）.                                                                                      如果我们把一个任务都包含这三个环节的话，可以理解为每个任务都有这三个环节的配置，配置完成后，系统自动化运行。 对于这样一个业务流程  ，我们需要思考：1.任务是贯串全过程的概念；2.任务是有状态的；3.任务有CRUD过程；4.任务的自动运行其实是有个调度机制的；5.如果一个任务 一个任务的配置，配置的工作量会很大，很麻烦，所以，我们会对配置进行分组，例如：创建一个配置设置，把一组任务和这个配置关联起来，简化 每个任务的配置，这个等于是我们又引入了一个概念，配置的管理，再引申一下，就是我们有合成的配置（coze配置，ffmpeg本地配置，缺省配置-  没有配置的就都走这个配置），我们有上传的配置，可能就是一组规则，例如，没有关联账号的任务，就自动寻找账号上传，关联了账号的，就用特 定的账号上传，上传完毕一个，sleep多久，在什么时间开始上传，看起来这是一个调度规则，在什么时间，对哪个任务做什么。                  我们先思考这个，先对业务建模。对于上传，可能的配置就是一组规则，把哪些任务和哪个规则关联起来。综合起来，我们就有了任务的概念，配置 的概念，规则的概念。 深入分析一下，作为业务流程重构的专家，按照最佳实践，我们思考这个问题
