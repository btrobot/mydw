# 应用登录 vs 业务账号连接 对照表

本文用于区分两条经常被混淆的链路：

1. **应用登录（Auth）**
2. **业务账号连接（Accounts Connect）**

---

## 1. 快速结论

| 项目 | 应用登录（Auth） | 业务账号连接（Accounts Connect） |
| --- | --- | --- |
| 本质 | 桌面应用授权 | 业务账号能力接入 |
| 目标 | 让操作者进入应用 | 让某个业务账号可用于发布/预览/导入导出 |
| 是否是系统门禁 | 是 | 否 |
| 是否依赖远端认证系统 | 是 | 不同，走业务账号连接逻辑 |
| 是否绑定本机 device_id | 是 | 不以 Auth device_id 为核心 |
| 是否决定路由放行 | 是 | 否 |
| 是否决定全局 Header 授权状态 | 是 | 否 |

---

## 2. 详细对照表

| 维度 | 应用登录（Auth） | 业务账号连接（Accounts Connect） |
| --- | --- | --- |
| 主要前端入口 | `frontend/src/features/auth/LoginPage.tsx` | `frontend/src/pages/Account.tsx` / 连接相关组件 |
| 主要前端路由 | `/login`、`/auth/*` | `/account` |
| 主要前端守卫 | `AuthRouteGate.tsx` | 无独立全局门禁，受应用登录放行后再使用 |
| 主要后端接口 | `/api/auth/*` | `/api/accounts/connect/*` |
| 历史兼容接口 | 无 | `/api/accounts/login/*`、`/api/accounts/logout/*`（deprecated） |
| 主要后端文件 | `backend/api/auth.py` | `backend/api/account.py` |
| 主要编排层 | `backend/services/auth_service.py` | 账号连接/浏览器会话相关逻辑 |
| 认证对象 | 操作者 + 当前机器 | 某个业务平台账号 |
| 关键状态 | `authenticated_active` / `grace` / `revoked` / `device_mismatch` / `expired` | 连接中 / 已连接 / 待验证码 / session 失效 等业务连接状态 |
| 关键持久化对象 | 本地 machine session + token + device_id | 业务账号 session / storage state / 浏览器上下文等 |
| 失败后的影响 | 不能进入或继续使用主应用 | 该业务账号不可用，但应用本身仍可进入 |
| UI 心智模型 | 系统门禁 | 业务资源连接器 |

---

## 3. 两条链路的正确顺序

```text
应用登录（Auth）
    ↓
进入主界面
    ↓
账号管理（Accounts）
    ↓
连接业务账号
    ↓
执行任务 / 发布 / 预览 / 导出导入 session
```

结论：

> **应用登录是前置条件，业务账号连接是登录后的业务能力。**

---

## 4. 如果把它们混在一起，会出现什么误解

### 误解 1：认为“登录页就是得物账号登录”

不对。

当前 `/login` 是**应用登录**，不是业务账号连接入口。  
业务账号连接发生在应用内部的账号管理模块里。

---

### 误解 2：认为“只要应用登录成功，就说明业务账号也能发”

不对。

应用登录成功只说明：

- 这台机器的本地授权有效
- 你可以进入应用

但不说明：

- 某个业务账号已经连好
- 业务账号 session 可用
- 业务账号当前能发布

---

### 误解 3：认为“业务账号失效 = 应用登录失效”

也不对。

业务账号连接失败，只会影响该账号相关业务能力；  
应用登录仍可能保持正常。

---

## 5. 用一句更准确的话描述当前系统

当前系统不是：

> “一个登录页 + 一些业务页面”

而是：

> “一个本地机器授权壳（Auth）+ 一套业务账号连接系统（Accounts Connect）+ 上层任务/素材/作品业务”

---

## 6. 讨论时建议统一的术语

为了避免后续沟通再次混淆，建议统一使用下面的叫法：

### 术语 A：应用登录

指：

- `/api/auth/*`
- 登录页
- Auth 状态页
- 本地 machine session
- 路由守卫

一句话：

> **操作者是否可以进入这台机器上的应用**

---

### 术语 B：业务账号连接

指：

- `/api/accounts/connect/*`
- 账号页里的连接/验证码/session 导入导出

一句话：

> **应用里的某个业务账号是否已经连好并可被使用**

---

## 7. 最终结论

这两套能力都和“登录”有关，但它们不是同一个问题：

- **应用登录**解决“谁可以进入应用”
- **业务账号连接**解决“进入应用后，哪些业务账号可以被使用”

如果把这两个层次分开看，当前架构就会清晰很多。
