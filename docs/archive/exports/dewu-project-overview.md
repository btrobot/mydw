# 得物掘金工具项目概览

## 1. 项目定位

这是一个面向得物平台内容运营的桌面自动化工具，目标是把账号管理、素材整理、商品/话题编排、视频处理、任务调度、自动发布放到一个统一工作台里。

根说明对它的定义很直接：这是一个“得物平台自动化视频发布系统”，支持多账号、素材管理、AI 剪辑和定时发布。

## 2. 当前技术栈

- 桌面壳：Electron
- 前端：React + TypeScript + Vite + Ant Design
- 前端数据层：React Query + OpenAPI 生成客户端 + Axios
- 后端：FastAPI + SQLAlchemy + SQLite
- 浏览器自动化：Patchright / Playwright
- 视频处理：FFmpeg / ffprobe
- 其他关键能力：AES 加密、Loguru 日志、Coze 工作流集成

## 3. 项目本质

这个项目已经不是单纯的“发视频脚本”或“接口调试工具”，而是一个本地桌面化的运营工作台。它把下面这些能力放到了同一条业务链上：

- 账号连接与会话复用
- 素材中心管理
- 商品与话题编排
- 任务批量创建和状态追踪
- 视频合成 / AI 工作流
- 调度上传和结果回写

## 4. 仓库主要组成

### `frontend/`

Electron + React 前端，也是用户实际看到的运营工作台。

主要模块包括：

- 数据看板
- 账号管理
- 任务管理
- 素材中心
- AI 剪辑
- 调度配置
- 配置档管理

### `backend/`

FastAPI 后端，负责业务 API、数据库、调度器、任务组装、自动化发布、合成流程和系统状态管理。

主要模块包括：

- `api/`：路由层
- `services/`：业务逻辑层
- `core/`：浏览器、得物客户端、配置、Coze 集成
- `models/`：数据库模型
- `migrations/`：SQLite 迁移脚本
- `tests/`：后端测试

### `docs/`、`design/`、`dev-docs/`

包含需求、架构设计、页面规格、ADR、迭代计划等文档，说明这个项目已经进入较系统化的工程维护状态，而不是一次性原型。

### `frontend/scripts/cli/`

一个用于调试和测试 API 的 CLI 工具，可以做健康检查、账号接口测试、任务/素材接口测试。

## 5. 运行方式

这是一个“桌面壳 + 本地后端”的组合应用：

1. Electron 启动桌面窗口
2. Electron 主进程拉起本地 FastAPI 后端
3. React 前端通过本地 HTTP API 与后端通信
4. 后端再去操作 SQLite、Patchright、FFmpeg、Coze 和得物创作者平台

## 6. 用户视角下它能做什么

- 管多个得物账号，并保存加密登录态
- 管视频、文案、封面、音频、话题、商品等素材
- 用素材批量组装发布任务
- 根据配置档决定任务是否需要先合成视频
- 按调度规则自动发到得物
- 对失败任务做重试、取消、编辑后重提

## 7. 目前能看到的一个小问题

文档版本和代码版本目前有轻微不同步：

- 根 README 标成 `0.2.0`
- 前端 `package.json` 和后端应用入口仍是 `0.1.0`

这不影响理解架构，但说明版本标识还需要统一。
