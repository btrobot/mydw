## 1. 系统概述

### 1.1 项目简介

**得物掘金工具 (DewuGoJin)** 是一款得物平台自动化视频发布系统，帮助用户管理多个得物账号、批量发布视频内容。

### 1.2 核心概念澄清

#### ⚠️ 重要：两类不同的"账号"和"登录"

| 概念 | 英文术语 | 说明 | 状态 |
|-----|---------|------|------|
| **系统用户** | System User | 多用户权限系统 | **暂不考虑** |
| **系统认证** | System Auth | 系统用户的登录认证 | **暂不考虑** |
| **得物账号** | Dewu Account | 得物平台账号 | **核心功能** |
| **得物连接** | Dewu Connection | 后端与得物平台的连接状态 | **核心功能** |

#### 术语对照表

| 旧术语（混淆） | 新术语（清晰） | 英文 |
|---------------|---------------|------|
| 系统登录 | ~~系统登录~~ | ~~System Login~~ |
| 账号登录 | 建立连接 | Establish Connection |
| 登出 | 断开连接 | Close Connection |
| 登录状态 | 连接状态 | Connection Status |
| 登录按钮 | 连接按钮 | Connect Button |

### 1.3 核心功能

| 功能模块 | 描述 |
|---------|------|
| 账号管理 | 得物账号的增删改查、连接状态监控、会话持久化 |
| 任务调度 | 定时发布、优先级队列、批量操作 |
| 素材管理 | 视频/文案/话题/封面管理 |
| AI 剪辑 | 视频智能剪辑、字幕生成 |
| 发布控制 | 定时策略、发布频率控制 |

### 6.1 敏感数据加密

```
┌─────────────────────────────────────────────────────────────┐
│                   敏感数据加密流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   用户敏感数据                                               │
│   ┌─────────────┐                                          │

│   │ State       │                                          │
│   └──────┬──────┘                                          │
│          │                                                  │
│          ▼                                                  │
│   ┌─────────────────────────────────────┐                   │
│   │        AES-256-GCM 加密             │                   │
│   │  ┌─────────────────────────────┐    │                   │
│   │  │ Key: PBKDF2HMAC(密码, 盐)   │    │                   │

```

### 6.2 安全措施清单

| 措施 | 实现 | 说明 |
|-----|------|------|
| 传输加密 | HTTPS | 生产环境必须启用 |
| 敏感数据加密 | AES-256-GCM | Cookie、Storage State |
| 密钥派生 | PBKDF2HMAC | 100000+ 迭代 |
| 跨域限制 | CORS | 白名单配置 |
| 输入验证 | Pydantic | 所有 API 参数 |
| SQL 注入防护 | SQLAlchemy ORM | 参数化查询 |
| XSS 防护 | React 默认转义 | 禁止 innerHTML |

### 9.2 协作流程

```
1. 需求分析
   用户 → Tech Lead (理解需求，分解任务)

2. 任务委派
   Tech Lead → Frontend Lead (前端任务)
   Tech Lead → Backend Lead (后端任务)
   Tech Lead → QA Lead (测试任务)

3. 并行开发
   Frontend ←→ Backend (API 契约协商)
              ↓
   Backend + Automation (自动化实现)

4. 质量把控
   QA Lead → 代码审查
   QA Lead → E2E 测试
   QA Lead → 安全扫描

5. 集成验证
   所有 Lead → 联合测试
   Tech Lead → 发布决策
```
