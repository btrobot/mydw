  技术方案建议

  方案 A: Server-Sent Events (SSE) ⭐ 推荐

  前端                    后端                      得物
    │                      │                         │
    │──── POST /login ────>│                         │
    │                      │──── GET login page ────>│
    │                      │<─── QR Code ────────────│
    │<─── EventStream ─────│                         │
    │   (QR code image)    │                         │
    │                      │    [User scans QR]      │
    │                      │──── Check status ───────>│
    │<─── Event: pending ──│<─── [Scanning...] ─────│
    │<─── Event: success ──│<─── Logged in ─────────│

  优点: 简单、单向推送、浏览器原生支持

  方案 B: WebSocket

  优点: 双向通信、更灵活
  缺点: 更复杂、需要心跳维护

● 登录得物功能任务分解（更新版）

  功能概述

  用户通过手机号 + 验证码方式登录得物创作者中心 (creator.dewu.com/login)，系统自动保存登录状态，支持多账号管理。

  登录流程

  ┌─────────┐    1. 输入手机号     ┌─────────────┐
  │  用户   │ ─────────────────────> │  登录弹窗   │
  └─────────┘                       └──────┬──────┘
       ▲                                    │
       │ 4. 显示验证码输入框                 │ 2. 点击获取验证码
       │                                    ▼
       │                           ┌─────────────────┐
       │                           │  后端发送短信   │
       │                           │  返回 success   │
       │                           └─────────────────┘
       │                                    │
       │ 5. 输入验证码        ┌──────┴──────┐
       │ ─────────────────────>│  验证并登录  │
       │                      └──────┬──────┘
       │                           │ 6. Playwright 填写验证码
       │                           │    并提交
       │                           ▼
       │                    ┌─────────────────┐
       └────────────────────│  登录成功       │
         7. 更新账号状态    │  保存 Storage   │
                            └─────────────────┘

  任务分解

  后端任务

  BE-LOGIN-01: 得物客户端 - 手机验证码登录

  描述: 重构 DewuClient 的 login 方法，支持手机号+验证码登录

  验收标准:
  - 输入手机号
  - 点击获取验证码按钮
  - 等待并输入验证码
  - 提交表单
  - 检测登录成功元素
  - 超时处理（验证码过期等）

  估计: 1d
  负责人: Automation Developer
  类型: backend
  依赖: -

  ---
  BE-LOGIN-02: 登录流程异步化

  描述: 将同步阻塞的登录改为异步模式，支持状态轮询

  验收标准:
  - POST /api/accounts/{id}/login/start - 启动登录，立即返回
  - GET /api/accounts/{id}/login/status - 轮询登录状态
  - 返回状态：pending, waiting_code, verifying, success, failed
  - 失败时返回错误原因

  估计: 0.5d
  负责人: Backend Lead
  类型: backend
  依赖: BE-LOGIN-01

  ---
  BE-LOGIN-03: 登录状态 SSE 推送

  描述: 通过 Server-Sent Events 主动推送登录状态

  验收标准:
  - GET /api/accounts/{id}/login/stream - SSE 连接
  - 推送事件：code_sent, waiting_verify, success, failed
  - 前端可直接接收，无需轮询

  估计: 0.5d
  负责人: Backend Lead
  类型: backend
  依赖: BE-LOGIN-02
